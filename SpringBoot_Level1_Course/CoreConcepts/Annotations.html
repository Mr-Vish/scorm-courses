Spring Boot Annotations

What are Annotations?

Definition: Annotations in Java are metadata that provide information about the program but are not part of the program itself. In Spring Boot, annotations are used to configure beans, define components, map HTTP requests, and inject dependencies without requiring XML configuration files.

Annotations use the @ symbol followed by the annotation name and can be applied to classes, methods, fields, or parameters. They help Spring Boot understand how to configure and wire your application components automatically.

Core Application Annotations

@SpringBootApplication

Definition: A convenience annotation that combines three essential Spring annotations into one. It serves as the entry point for Spring Boot applications.

Components:

  • @Configuration: Indicates that the class contains Spring configuration and can define beans using @Bean methods
  • @EnableAutoConfiguration: Tells Spring Boot to automatically configure beans based on classpath dependencies
  • @ComponentScan: Enables automatic discovery and registration of Spring components in the current package and sub-packages
@SpringBootApplication
public class MyApplication {
    public static void main(String[] args) {
        SpringApplication.run(MyApplication.class, args);
    }
}

Web Layer Annotations

@RestController

Definition: A specialized version of @Controller that combines @Controller and @ResponseBody. It indicates that the class handles HTTP requests and returns data directly in the response body (typically JSON or XML) rather than rendering a view.

@RestController
@RequestMapping("/api")
public class UserController {
    // All methods return data directly to HTTP response
}

@Controller

Definition: Marks a class as a Spring MVC controller that handles HTTP requests and typically returns view names for rendering HTML pages.

@RequestMapping

Definition: Maps HTTP requests to specific handler methods or classes. It can specify the URL path, HTTP method, request parameters, and headers.

@RequestMapping(value = "/users", method = RequestMethod.GET)
public List<User> getAllUsers() {
    return userService.findAll();
}

HTTP Method-Specific Annotations

Definition: Specialized annotations that combine @RequestMapping with specific HTTP methods for cleaner, more readable code.

  • @GetMapping: Shortcut for @RequestMapping(method = RequestMethod.GET) - used for retrieving data
  • @PostMapping: Shortcut for @RequestMapping(method = RequestMethod.POST) - used for creating new resources
  • @PutMapping: Shortcut for @RequestMapping(method = RequestMethod.PUT) - used for updating existing resources
  • @DeleteMapping: Shortcut for @RequestMapping(method = RequestMethod.DELETE) - used for deleting resources
  • @PatchMapping: Shortcut for @RequestMapping(method = RequestMethod.PATCH) - used for partial updates
@GetMapping("/users/{id}")
public User getUserById(@PathVariable Long id) {
    return userService.findById(id);
}

@PostMapping("/users")
public User createUser(@RequestBody User user) {
    return userService.save(user);
}

Dependency Injection Annotations

@Autowired

Definition: Enables automatic dependency injection by type. Spring automatically finds and injects the appropriate bean into the annotated field, constructor, or method parameter.

@Service
public class UserController {
    @Autowired
    private UserService userService; // Field injection

    // Constructor injection (recommended)
    public UserController(UserService userService) {
        this.userService = userService;
    }
}

Component Stereotype Annotations

Definition: Annotations that mark classes as Spring-managed components with specific roles in the application architecture.

@Component

Definition: Generic stereotype annotation that marks a class as a Spring-managed component. Other stereotype annotations are specializations of @Component.

@Service

Definition: Marks a class as a service component that contains business logic. It's a specialization of @Component for the service layer.

@Service
public class UserService {
    public User findById(Long id) {
        // Business logic implementation
    }
}

@Repository

Definition: Marks a class as a data access component (DAO). It's a specialization of @Component for the persistence layer and provides automatic exception translation.

@Repository
public class UserRepository {
    public User save(User user) {
        // Data access logic
    }
}

Configuration and Property Annotations

@Value

Definition: Injects values from properties files, environment variables, or system properties into fields or method parameters.

@Value("${app.name:Default App}")
private String appName; // Injects app.name property with default value

@Value("${server.port}")
private int serverPort;

@ConfigurationProperties

Definition: Binds external configuration properties to a Java object, providing type-safe configuration binding for complex property structures.

@ConfigurationProperties(prefix = "app")
@Component
public class AppProperties {
    private String name;
    private String version;
    private Database database = new Database();

    // getters and setters

    public static class Database {
        private String url;
        private String username;
        // getters and setters
    }
}

Request Parameter Annotations

Definition: Annotations used to extract data from HTTP requests in controller methods.

  • @PathVariable: Extracts values from URI path segments
  • @RequestParam: Extracts query parameters from the request URL
  • @RequestBody: Binds the HTTP request body to a method parameter
  • @RequestHeader: Extracts values from HTTP headers
@GetMapping("/users/{id}")
public User getUser(@PathVariable Long id,
                    @RequestParam(defaultValue = "false") boolean includeDetails) {
    return userService.findById(id, includeDetails);
}

@PostMapping("/users")
public User createUser(@RequestBody User user,
                       @RequestHeader("Authorization") String authToken) {
    return userService.create(user, authToken);
}
