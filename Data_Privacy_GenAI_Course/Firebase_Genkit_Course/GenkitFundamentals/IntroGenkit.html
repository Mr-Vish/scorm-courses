Introduction to Firebase Genkit

What Is Firebase Genkit?

Firebase Genkit is an open-source framework developed by Google for building AI-powered applications. It provides a unified, type-safe interface for working with multiple AI models, structured generation, retrieval systems, and evaluation tools. Genkit abstracts the complexity of integrating various LLM providers while maintaining flexibility and observability.

Why Genkit?

  • Provider Agnostic: Switch between AI providers (Gemini, GPT, Claude, Ollama) without rewriting code
  • Type Safety: Full TypeScript support with Zod schema validation for inputs and outputs
  • Built-in Observability: Automatic tracing, logging, and debugging of AI interactions
  • Production Ready: Designed for deployment to Firebase Cloud Functions, Cloud Run, or any Node.js environment
  • Developer Experience: Visual developer UI for testing flows, inspecting traces, and evaluating outputs
  • Extensible: Plugin architecture for custom models, retrievers, and tools

Core Architecture Components

Flows

Flows are the fundamental building blocks in Genkit. They are type-safe, observable functions that orchestrate AI operations. Each flow has:

  • Defined input and output schemas (using Zod)
  • Automatic tracing and logging
  • Error handling and retry logic
  • Ability to call AI models, tools, and other flows

Models

Models provide an abstraction layer over different LLM providers. Genkit supports:

  • Google AI: Gemini 1.5 Flash, Gemini 1.5 Pro, Gemini 2.0
  • OpenAI: GPT-4, GPT-4 Turbo, GPT-3.5
  • Anthropic: Claude 3 Opus, Claude 3 Sonnet, Claude 3 Haiku
  • Ollama: Local models (Llama, Mistral, etc.)
  • Vertex AI: Enterprise-grade Google Cloud models

Prompts

Prompts are managed templates with input/output schemas. They enable:

  • Reusable prompt definitions across your application
  • Version control and testing of prompts
  • Input validation and output parsing
  • Separation of prompt engineering from application logic

Retrievers

Retrievers enable retrieval-augmented generation (RAG) by connecting to vector databases:

  • Pinecone, Chroma, Weaviate integration
  • Document chunking and embedding generation
  • Semantic search capabilities
  • Context injection into prompts

Evaluators

Evaluators assess AI output quality using automated metrics:

  • Faithfulness (accuracy to source material)
  • Answer relevance
  • Maliciousness detection
  • Custom evaluation criteria

Installation and Setup

Step 1: Install Genkit

# Install core Genkit package
npm install genkit

# Install AI provider plugin (example: Google AI)
npm install @genkit-ai/googleai

# Install development tools
npm install -D genkit-cli

Step 2: Initialize Genkit

import { genkit } from 'genkit';
import { googleAI, gemini15Flash } from '@genkit-ai/googleai';

// Configure Genkit with plugins and default model
const ai = genkit({
  plugins: [googleAI()],
  model: gemini15Flash,
});

Step 3: Set Up API Keys

Configure environment variables for your chosen AI provider:

# .env file
GOOGLE_GENAI_API_KEY=your_google_api_key
OPENAI_API_KEY=your_openai_api_key
ANTHROPIC_API_KEY=your_anthropic_api_key

Genkit Developer UI

Genkit includes a powerful visual developer interface for testing and debugging:

# Start the developer UI
npx genkit start

# Access at http://localhost:4000

The Developer UI provides:

  • Flow Runner: Test flows with custom inputs
  • Trace Inspector: View detailed execution traces with timing and token usage
  • Prompt Editor: Design and test prompts interactively
  • Evaluator Dashboard: Run and visualize evaluation results
