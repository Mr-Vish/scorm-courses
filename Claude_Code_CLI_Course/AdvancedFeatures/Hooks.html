Hooks System

What Are Hooks?

Hooks are shell commands that execute automatically in response to specific Claude Code events. They allow you to customize and extend Claude Code's behavior without modifying the tool itself.

Hook Types

HookTriggerUse Case
PreToolUseBefore a tool executesValidate, block, or modify tool calls
PostToolUseAfter a tool executesPost-processing, logging, notifications
NotificationWhen Claude wants to notify the userCustom notifications (Slack, email, etc.)
StopWhen Claude finishes a turnAuto-validation, test running

Configuration Example

Add hooks to your .claude/settings.json:

{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Edit",
        "command": "node validate-edit.js"
      }
    ],
    "PostToolUse": [
      {
        "matcher": "Write",
        "command": "npx prettier --write $CLAUDE_FILE_PATH"
      }
    ]
  }
}

Practical Hook Examples

  • Auto-format: Run Prettier/ESLint after every file write
  • Security scan: Check for secrets before committing
  • Test runner: Automatically run related tests after code changes
  • Notification: Send Slack messages when long tasks complete
