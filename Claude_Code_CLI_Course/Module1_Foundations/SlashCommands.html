Command Structure and Slash Commands

Understanding the Command Interface

While Claude Code CLI primarily operates through natural language conversation, it also provides structured commands called "slash commands" that offer quick access to specific functionality. Understanding both interaction modes—conversational and command-based—enables efficient and effective use of the tool.

Dual Interaction Paradigm

Claude Code CLI supports two complementary interaction styles:

Conversational Mode (Primary):

  • Natural language requests and questions
  • Flexible expression of intent without memorizing syntax
  • Contextual understanding based on conversation history
  • Suitable for complex, multi-step tasks and exploration

Command Mode (Slash Commands):

  • Structured commands with predictable behavior
  • Quick access to frequently used operations
  • Explicit control over specific functionality
  • Suitable for routine tasks and workflow integration

Slash Command Fundamentals

Slash commands are special instructions that begin with a forward slash (/) and trigger specific behaviors within Claude Code CLI. They provide a structured alternative to natural language for common operations.

Command Syntax Structure

Slash commands follow a consistent syntax pattern:

/command [arguments] [options] Examples: /help /review src/main.ts /clear --confirm

Syntax Components:

  • Command Name: The primary action to perform (e.g., /help, /review)
  • Arguments: Required or optional parameters (e.g., file paths, search terms)
  • Options/Flags: Modifiers that change command behavior (e.g., --confirm, --verbose)

Command Discovery and Help

Claude Code CLI provides built-in mechanisms for discovering and learning about available commands:

# List all available slash commands /help # Get detailed help for a specific command /help review # Show command examples /help review --examples

Core Slash Commands

The following slash commands provide essential functionality for common development tasks.

/help - Documentation and Guidance

Purpose: Access documentation, command reference, and usage examples

Usage Patterns:

# General help /help # Command-specific help /help [command-name] # Show examples /help [command-name] --examples # Search help topics /help --search "git operations"

Use Cases:

  • Learning available commands and their syntax
  • Understanding command options and parameters
  • Finding examples for specific tasks
  • Troubleshooting command usage issues

/review - Code Review and Analysis

Purpose: Perform code review, identify issues, and suggest improvements

Usage Patterns:

# Review specific file /review src/components/UserProfile.tsx # Review directory /review src/components/ # Review with focus areas /review src/api/ --focus security,performance # Review recent changes /review --git-diff

Review Aspects:

  • Code Quality: Identifies code smells, anti-patterns, and maintainability issues
  • Security: Detects potential security vulnerabilities and unsafe practices
  • Performance: Highlights performance bottlenecks and optimization opportunities
  • Best Practices: Suggests adherence to language-specific conventions and patterns
  • Documentation: Evaluates code documentation and suggests improvements

/clear - Context Management

Purpose: Clear conversation history and reset context

Usage Patterns:

# Clear conversation history /clear # Clear with confirmation prompt /clear --confirm # Clear and reset project index /clear --full

When to Use:

  • Starting a new task unrelated to previous conversation
  • Resolving context confusion or incorrect assumptions
  • Freeing up context window for new information
  • Troubleshooting unexpected behavior

/doctor - Diagnostic and Troubleshooting

Purpose: Diagnose configuration issues and system health

Usage Patterns:

# Run full diagnostic check /doctor # Check specific subsystem /doctor --check auth /doctor --check config /doctor --check permissions # Verbose output for debugging /doctor --verbose

Diagnostic Checks:

  • Authentication: Verifies API key validity and connectivity
  • Configuration: Validates configuration file syntax and values
  • Permissions: Checks file system and command execution permissions
  • Dependencies: Verifies required tools and libraries are available
  • Performance: Identifies performance issues or bottlenecks

/diff - Change Comparison

Purpose: Compare file versions and review changes

Usage Patterns:

# Show git diff for current changes /diff # Compare specific files /diff src/old.ts src/new.ts # Show diff for specific commit /diff --commit abc123 # Compare branches /diff --branch main feature/new-api

Diff Analysis Capabilities:

  • Highlights added, modified, and deleted lines
  • Explains the purpose and impact of changes
  • Identifies potential issues introduced by changes
  • Suggests improvements or alternative approaches

/test - Testing Operations

Purpose: Run tests and analyze test results

Usage Patterns:

# Run all tests /test # Run specific test file /test tests/user.test.ts # Run tests matching pattern /test --pattern "User*" # Run with coverage /test --coverage

Test Analysis Features:

  • Executes test suites and captures results
  • Analyzes test failures and suggests fixes
  • Evaluates test coverage and identifies gaps
  • Recommends additional test cases

/commit - Git Commit Assistance

Purpose: Generate commit messages and manage git operations

Usage Patterns:

# Generate commit message for staged changes /commit # Generate and commit automatically /commit --auto # Review changes before committing /commit --review # Generate conventional commit message /commit --conventional

Commit Message Generation:

  • Analyzes staged changes to understand modifications
  • Generates descriptive, conventional commit messages
  • Follows project-specific commit conventions
  • Includes relevant context and reasoning

Advanced Slash Commands

Beyond core commands, Claude Code CLI provides advanced commands for specialized tasks.

/refactor - Code Refactoring

Purpose: Perform structured code refactoring operations

# Refactor specific function /refactor extractMethod src/utils.ts:45-67 # Rename symbol across project /refactor rename OldClassName NewClassName # Apply design pattern /refactor applyPattern strategy src/payment/

/explain - Code Explanation

Purpose: Generate detailed explanations of code

# Explain specific file /explain src/algorithm.ts # Explain function or class /explain src/utils.ts:calculateTotal # Explain with diagrams /explain src/architecture/ --diagram

/search - Codebase Search

Purpose: Search codebase with semantic understanding

# Search for functionality /search "user authentication logic" # Search by pattern /search --regex "function.*User.*" # Search in specific directory /search "API calls" --path src/services/

/generate - Code Generation

Purpose: Generate boilerplate code and scaffolding

# Generate component /generate component UserProfile --type react # Generate test file /generate test src/utils.ts # Generate API endpoint /generate api /users --crud

Command Composition and Chaining

Slash commands can be combined with natural language to create powerful workflows.

Sequential Command Execution

Multiple commands can be executed in sequence:

# Review, then test, then commit /review src/api/ /test tests/api/ /commit --auto

Command with Natural Language Context

Commands can be enhanced with conversational context:

# Command with explanation /review src/payment.ts Focus on security vulnerabilities, especially around payment processing and data validation. # Command with follow-up /refactor extractMethod src/utils.ts:45-67 Name the new method calculateDiscountedPrice and add JSDoc documentation.

Custom Command Configuration

Projects can define custom slash commands for team-specific workflows.

Defining Custom Commands

Custom commands are defined in .claude/commands.json:

{ "commands": { "/deploy-staging": { "description": "Deploy to staging environment", "steps": [ "npm run build", "npm run test", "git push staging main" ] }, "/check-quality": { "description": "Run full quality checks", "steps": [ "/test --coverage", "/review src/ --focus security,performance", "npm run lint" ] } } }

Command Aliases

Create shortcuts for frequently used command patterns:

{ "aliases": { "/r": "/review", "/t": "/test", "/c": "/commit", "/qa": "/check-quality" } }

Command Best Practices

Effective use of slash commands requires understanding when and how to use them appropriately.

When to Use Slash Commands

  • Routine Operations: Frequently performed tasks with predictable requirements
  • Workflow Integration: Standardized steps in development workflows
  • Quick Actions: Simple operations that don't require explanation
  • Automation: Tasks that will be scripted or automated

When to Use Natural Language

  • Complex Requests: Multi-step tasks requiring decision-making
  • Exploration: Investigating unfamiliar code or concepts
  • Contextual Tasks: Operations requiring understanding of broader context
  • Learning: When you're unsure of the exact approach needed

Combining Both Approaches

The most effective use of Claude Code CLI often combines both interaction modes:

# Start with natural language for context "I need to refactor the payment processing module to improve testability" # Use commands for specific operations /review src/payment/ /refactor extractMethod src/payment/processor.ts:89-145 /generate test src/payment/processor.ts # Return to natural language for guidance "Now help me update the integration tests to work with the refactored code"

Command Output and Feedback

Understanding command output helps interpret results and take appropriate action.

Output Formats

Commands provide output in various formats:

  • Text Summary: Human-readable description of results
  • Structured Data: JSON or table format for programmatic use
  • Diff Format: Side-by-side or unified diff for changes
  • Interactive Prompts: Questions requiring user input

Status Indicators

Commands use consistent status indicators:

Indicator Meaning Action Required
✓ Success Operation completed successfully None, proceed to next step
⚠ Warning Operation completed with concerns Review warnings, may need attention
✗ Error Operation failed Address error before proceeding
? Prompt User input required Provide requested information

Key Takeaways

  • Claude Code CLI supports both natural language conversation and structured slash commands
  • Slash commands provide quick access to common operations with predictable behavior
  • Core commands include /help, /review, /clear, /doctor, /diff, /test, and /commit
  • Advanced commands support refactoring, explanation, search, and code generation
  • Commands can be combined with natural language for powerful workflows
  • Custom commands and aliases enable team-specific workflow optimization
  • Effective use involves choosing the right interaction mode for each task
  • Command output uses consistent formats and status indicators for clarity
