Installation and Environment Configuration

System Requirements and Prerequisites

Before installing Claude Code CLI, it's essential to understand the system requirements and ensure your development environment meets the necessary prerequisites. This preparation phase is critical for smooth operation and optimal performance.

Operating System Compatibility

Claude Code CLI is designed to work across major operating systems, but with platform-specific considerations:

Platform Support Level Requirements Considerations
macOS Full Native Support macOS 10.15 (Catalina) or later Recommended platform; best performance and compatibility
Linux Full Native Support Modern distributions (Ubuntu 20.04+, Fedora 34+, etc.) Excellent performance; may require additional package dependencies
Windows Via WSL2 Windows 10 version 2004+ or Windows 11 with WSL2 enabled Requires Windows Subsystem for Linux 2; performance comparable to native Linux

Runtime Dependencies

Claude Code CLI requires specific runtime environments to function correctly:

Node.js Environment

  • Version Requirement: Node.js 18.0.0 or higher (LTS versions recommended)
  • Package Manager: npm (bundled with Node.js) or yarn
  • Rationale: Claude Code CLI is built on Node.js for cross-platform compatibility and access to the rich npm ecosystem
  • Verification: Run node --version and npm --version to confirm installation

Terminal Environment

  • Shell Compatibility: bash, zsh, fish, or other POSIX-compliant shells
  • Terminal Emulator: Any modern terminal (Terminal.app, iTerm2, GNOME Terminal, Windows Terminal, etc.)
  • Character Encoding: UTF-8 support for proper display of special characters and emojis

Authentication Requirements

Claude Code CLI requires authentication with Anthropic's services through one of two methods:

Option 1: API Key Authentication

  • Obtain an API key from the Anthropic Console (console.anthropic.com)
  • Suitable for individual developers and programmatic access
  • Provides usage-based billing with rate limits based on tier
  • Requires secure storage of API credentials

Option 2: Claude Pro/Max Subscription

  • Use existing Claude Pro or Claude Max subscription credentials
  • Suitable for users already subscribed to Claude's web interface
  • Provides access within subscription limits
  • Simplified authentication through browser-based login

Installation Process

The installation of Claude Code CLI follows standard npm package installation procedures, but with important configuration steps that must be completed for proper operation.

Global Installation via npm

The recommended installation method uses npm's global package installation:

# Install Claude Code CLI globally npm install -g @anthropic-ai/claude-code # Verify installation claude --version

Installation Breakdown:

  • -g flag: Installs the package globally, making the claude command available system-wide
  • Package name: @anthropic-ai/claude-code is the official package identifier
  • Version verification: Confirms successful installation and displays the installed version

Alternative Installation Methods

Using yarn:

# Install using yarn package manager yarn global add @anthropic-ai/claude-code # Verify installation claude --version

Project-Local Installation (Not Recommended):

While technically possible, local installation is not recommended as Claude Code CLI is designed to operate across multiple projects:

# Local installation (not recommended) npm install @anthropic-ai/claude-code # Would require npx to run npx claude

Initial Configuration

After installation, Claude Code CLI requires initial configuration to establish authentication and set default preferences.

Authentication Setup

API Key Configuration:

# Set API key as environment variable export ANTHROPIC_API_KEY="your-api-key-here" # Or configure through Claude Code CLI claude config set api-key # Verify authentication claude auth status

Interactive Login (Pro/Max Subscribers):

# Initiate browser-based authentication claude login # Follow browser prompts to authenticate # Returns to terminal upon successful authentication

Configuration File Structure

Claude Code CLI uses a hierarchical configuration system with multiple levels of precedence:

Global Configuration (~/.claude/config.json):

System-wide settings that apply to all projects unless overridden:

{ "model": "claude-3-5-sonnet-20241022", "maxTokens": 4096, "temperature": 0.7, "autoApprove": { "read": false, "write": false, "execute": false }, "editor": "code", "theme": "dark" }

Project Configuration (.claude/config.json):

Project-specific settings that override global configuration:

{ "model": "claude-3-5-sonnet-20241022", "context": { "includePatterns": ["src/**/*.ts", "tests/**/*.ts"], "excludePatterns": ["node_modules/**", "dist/**"] }, "hooks": { "preCommit": "npm run lint", "postGenerate": "npm run format" } }

Environment Configuration

Proper environment configuration ensures Claude Code CLI operates efficiently and securely within your development workflow.

Environment Variables

Key environment variables that control Claude Code CLI behavior:

Variable Purpose Example Value
ANTHROPIC_API_KEY Authentication credential for API access sk-ant-api03-...
CLAUDE_MODEL Default model to use for requests claude-3-5-sonnet-20241022
CLAUDE_CONFIG_DIR Custom location for configuration files ~/.config/claude
CLAUDE_LOG_LEVEL Logging verbosity (error, warn, info, debug) info

Shell Integration

For optimal user experience, configure shell integration to enable features like command completion and aliases:

Bash Configuration (~/.bashrc):

# Claude Code CLI environment variables export ANTHROPIC_API_KEY="your-api-key" export CLAUDE_MODEL="claude-3-5-sonnet-20241022" # Enable command completion eval "$(claude completion bash)" # Useful aliases alias cc="claude" alias ccr="claude review"

Zsh Configuration (~/.zshrc):

# Claude Code CLI environment variables export ANTHROPIC_API_KEY="your-api-key" export CLAUDE_MODEL="claude-3-5-sonnet-20241022" # Enable command completion eval "$(claude completion zsh)" # Useful aliases alias cc="claude" alias ccr="claude review"

Workspace Initialization

When starting work on a new project with Claude Code CLI, proper workspace initialization ensures optimal performance and context awareness.

Project Initialization

# Navigate to project directory cd /path/to/your/project # Initialize Claude Code CLI for this project claude init # This creates .claude/ directory with default configuration

Initialization Process:

  • Creates .claude/ directory in project root
  • Generates default config.json with project-specific settings
  • Creates .claudeignore file (similar to .gitignore) to exclude files from context
  • Optionally creates CLAUDE.md file for project-specific instructions

The CLAUDE.md File

The CLAUDE.md file serves as a project-specific instruction manual for Claude Code CLI:

# Project: E-Commerce Platform ## Architecture Overview This is a microservices-based e-commerce platform using Node.js and TypeScript. ## Code Style Guidelines - Use TypeScript strict mode - Follow Airbnb style guide - Prefer functional programming patterns - Use async/await over promises ## Testing Requirements - Minimum 80% code coverage - Unit tests for all business logic - Integration tests for API endpoints ## Common Tasks - Run tests: `npm test` - Build: `npm run build` - Deploy: `npm run deploy:staging`

Verification and Testing

After installation and configuration, verify that Claude Code CLI is functioning correctly:

Basic Functionality Tests

# Check version and installation claude --version # Verify authentication claude auth status # Test basic query claude "What files are in the current directory?" # Test file reading capability claude "Summarize the README.md file" # Run diagnostic check claude doctor

The Doctor Command

The claude doctor command performs comprehensive system checks:

  • Verifies Node.js version compatibility
  • Checks authentication status and API connectivity
  • Validates configuration file syntax
  • Tests file system permissions
  • Identifies common configuration issues
  • Provides recommendations for optimization

Troubleshooting Common Installation Issues

Permission Errors

Issue: npm installation fails with EACCES permission errors

Solution: Configure npm to use a different directory or use a Node version manager (nvm)

# Using nvm (recommended) curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash nvm install 18 nvm use 18 npm install -g @anthropic-ai/claude-code

Command Not Found

Issue: claude command not recognized after installation

Solution: Ensure npm global bin directory is in PATH

# Check npm global bin location npm config get prefix # Add to PATH (add to ~/.bashrc or ~/.zshrc) export PATH="$PATH:$(npm config get prefix)/bin"

Authentication Failures

Issue: API key authentication fails

Solution: Verify API key format and permissions

  • Ensure API key starts with "sk-ant-api03-"
  • Check that API key has not expired
  • Verify API key has appropriate permissions in Anthropic Console
  • Ensure no extra whitespace in environment variable

Key Takeaways

  • Claude Code CLI requires Node.js 18+ and works across macOS, Linux, and Windows (via WSL2)
  • Installation is performed via npm global package installation
  • Authentication can be configured using API keys or Claude Pro/Max subscription
  • Configuration follows a hierarchical system with global and project-specific settings
  • Proper workspace initialization with claude init optimizes project-specific behavior
  • The CLAUDE.md file provides project-specific context and guidelines
  • The claude doctor command helps diagnose and resolve configuration issues
