Tools and Delegation

Equipping Agents with Tools

Tools give agents the ability to interact with the real world - search the web, read files, query databases, and call APIs:

from crewai import Agent
from crewai_tools import SerperDevTool, ScrapeWebsiteTool, FileReadTool

# Create tools
search_tool = SerperDevTool()      # Google search via Serper API
scrape_tool = ScrapeWebsiteTool()  # Web page scraping
file_tool = FileReadTool()         # Local file reading

# Assign tools to an agent
researcher = Agent(
    role="Research Analyst",
    goal="Find accurate, up-to-date information",
    backstory="Expert researcher with access to web search tools.",
    tools=[search_tool, scrape_tool, file_tool],
    llm="gpt-4o",
)

Creating Custom Tools

from crewai_tools import BaseTool
from pydantic import BaseModel, Field

class DatabaseQueryInput(BaseModel):
    query: str = Field(description="SQL query to execute")

class DatabaseQueryTool(BaseTool):
    name: str = "database_query"
    description: str = "Execute a SQL query against the analytics database"
    args_schema: type[BaseModel] = DatabaseQueryInput

    def _run(self, query: str) -> str:
        # Execute query against your database
        import sqlite3
        conn = sqlite3.connect("analytics.db")
        result = conn.execute(query).fetchall()
        conn.close()
        return str(result)

Delegation

When allow_delegation=True, an agent can ask other agents in the crew for help. This enables dynamic collaboration:

  • The researcher can ask the writer to rephrase findings for clarity
  • The writer can ask the researcher for additional data on a specific point
  • Delegation happens automatically through natural language requests between agents

Best Practices

  • Clear roles: Give agents distinct, non-overlapping responsibilities
  • Specific goals: Vague goals lead to vague outputs - be precise
  • Rich backstories: Backstories improve output quality by providing context
  • Limit tools: Only give agents the tools they need - too many tools confuse the model
  • Expected outputs: Define clear output formats in task descriptions
