Agents, Tasks, and Crews

What is CrewAI?

CrewAI is a Python framework for orchestrating multiple AI agents that collaborate like a team of specialists. Each agent has a defined role, goal, and backstory, making the system intuitive to design and debug. Think of it as building a virtual team where each member has a specific job.

Core Concepts

ConceptDescriptionReal-World Analogy
AgentAn autonomous unit with a role, goal, and toolsA team member with a job title
TaskA specific piece of work assigned to an agentA ticket or work item
CrewA group of agents working together on tasksA project team
ProcessHow tasks are executed (sequential or hierarchical)A workflow or methodology

Defining Agents

from crewai import Agent

researcher = Agent(
    role="Senior Research Analyst",
    goal="Find and analyze the latest AI industry trends",
    backstory="You are an experienced research analyst with 10 years in tech.",
    verbose=True,
    allow_delegation=True,
    llm="gpt-4o",
)

writer = Agent(
    role="Technical Writer",
    goal="Write clear, engaging articles about AI trends",
    backstory="You are a skilled writer who makes complex topics accessible.",
    verbose=True,
    allow_delegation=False,
    llm="gpt-4o",
)

Defining Tasks and Crews

from crewai import Task, Crew, Process

research_task = Task(
    description="Research the top 5 AI trends for 2025. Include market data.",
    expected_output="A detailed report with 5 trends, each with data and analysis.",
    agent=researcher,
)

writing_task = Task(
    description="Write a blog post based on the research report.",
    expected_output="A 1000-word blog post in markdown format.",
    agent=writer,
    context=[research_task],  # Gets output from research_task
)

# Assemble the crew
crew = Crew(
    agents=[researcher, writer],
    tasks=[research_task, writing_task],
    process=Process.sequential,  # Tasks run in order
    verbose=True,
)

# Kick off the work
result = crew.kickoff()
print(result)

Process Types

  • Sequential: Tasks execute one after another. Output of each task is available to the next. Best for linear workflows.
  • Hierarchical: A manager agent delegates tasks to team members. Best for complex projects needing coordination.
