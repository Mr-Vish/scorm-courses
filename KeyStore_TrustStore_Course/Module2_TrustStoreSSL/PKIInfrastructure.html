Certificate Authorities and PKI Infrastructure

Understanding Public Key Infrastructure (PKI)

Public Key Infrastructure (PKI) is a framework of policies, procedures, hardware, software, and people that create, manage, distribute, use, store, and revoke digital certificates. PKI enables secure electronic transfer of information for authentication, encryption, and digital signatures.

PKI Components

Component Role Responsibility
Certificate Authority (CA) Trust anchor Issues and signs digital certificates
Registration Authority (RA) Identity verification Verifies certificate requests before CA signs
Certificate Repository Storage Stores and distributes certificates and CRLs
Certificate Revocation List (CRL) Revocation tracking Lists revoked certificates
OCSP Responder Real-time validation Provides certificate status in real-time

Private Certificate Authorities

Internal CAs operated by organizations for their own infrastructure:

Advantages of Private CA

  • Cost Savings: No per-certificate fees
  • Control: Full control over certificate policies
  • Flexibility: Custom certificate attributes and validity periods
  • Speed: Instant certificate issuance
  • Internal Use: Ideal for internal systems and development

Challenges of Private CA

  • Trust Distribution: Must manually distribute root certificate
  • Management Overhead: Requires infrastructure and expertise
  • No Public Trust: Not trusted by external parties
  • Security Responsibility: Organization responsible for CA security

Certificate Validation Levels

Domain Validated (DV) Certificates

Validation: Proves control of domain name only

Issuance Time: Minutes to hours

Cost: Low ($0-$50/year)

Use Cases: Blogs, personal websites, development environments

Trust Level: Basic - only domain ownership verified

Organization Validated (OV) Certificates

Validation: Verifies organization identity and domain control

Issuance Time: 1-3 days

Cost: Medium ($50-$200/year)

Use Cases: Corporate websites, business applications

Trust Level: Medium - organization identity verified

Extended Validation (EV) Certificates

Validation: Rigorous verification of legal entity and operational existence

Issuance Time: 1-2 weeks

Cost: High ($200-$1000+/year)

Use Cases: Financial institutions, e-commerce, high-security sites

Trust Level: Highest - comprehensive verification

Certificate Revocation

Reasons for Certificate Revocation

  • Private Key Compromise: Private key exposed or stolen
  • CA Compromise: Issuing CA's security breached
  • Affiliation Changed: Organization name or ownership changed
  • Certificate Misuse: Certificate used improperly
  • Superseded: Replaced by newer certificate
  • Cessation of Operation: Service discontinued

Certificate Revocation List (CRL)

CRL is a list of revoked certificates published by the CA:

How It Works:
1. CA publishes CRL at regular intervals
2. Clients download CRL periodically
3. Clients check certificate serial number against CRL
4. If found in CRL, certificate is rejected

CRL Advantages

  • Simple to implement
  • Works offline once downloaded
  • Standardized format

CRL Disadvantages

  • Can become very large
  • Delayed revocation (until next CRL update)
  • Bandwidth intensive
  • Scalability issues

Online Certificate Status Protocol (OCSP)

OCSP provides real-time certificate status checking:

How It Works:
1. Client sends certificate serial number to OCSP responder
2. OCSP responder checks certificate status
3. Responder returns: Good, Revoked, or Unknown
4. Client accepts or rejects certificate based on response

OCSP Advantages

  • Real-time status information
  • Smaller data transfer than CRL
  • More current revocation information

OCSP Disadvantages

  • Requires online connectivity
  • Privacy concerns (CA knows which sites you visit)
  • Performance impact (additional network request)
  • OCSP responder availability dependency

OCSP Stapling

Optimization where server provides OCSP response during handshake:

Benefits:
- Reduces client latency
- Improves privacy
- Reduces load on OCSP responders
- Server caches OCSP response

Certificate Lifecycle in PKI

Certificate Lifecycle Stages

Stage Activities Duration
Request Generate CSR, submit to CA Minutes to hours
Validation CA verifies identity and domain Minutes to weeks (depends on validation level)
Issuance CA signs and issues certificate Immediate after validation
Active Use Certificate in production use Validity period (typically 1-2 years)
Renewal Request new certificate before expiration 30-60 days before expiry
Revocation Certificate invalidated if compromised Immediate when needed
Expiration Certificate no longer valid End of validity period

Selecting a Certificate Authority

Evaluation Criteria

For Public-Facing Applications

  • Browser Trust: Root certificate trusted by major browsers
  • Reputation: Established CA with good security track record
  • Support: Quality customer support and documentation
  • Pricing: Competitive pricing for required validation level
  • Features: Wildcard, SAN, multi-domain support as needed
  • Automation: API support for automated certificate management

For Internal Applications

  • Private CA: Consider internal CA for cost savings
  • Management Tools: Certificate lifecycle management capabilities
  • Integration: Integration with existing infrastructure
  • Scalability: Ability to handle certificate volume

PKI Best Practices

Organizational Policies

  • Certificate Policy (CP): Define rules for certificate issuance and use
  • Certificate Practice Statement (CPS): Document how policies are implemented
  • Key Management: Procedures for key generation, storage, and destruction
  • Audit Requirements: Regular audits of certificate usage and compliance

Operational Practices

  • Inventory Management: Maintain complete certificate inventory
  • Expiration Monitoring: Automated alerts for expiring certificates
  • Renewal Process: Defined procedures for certificate renewal
  • Incident Response: Plan for handling compromised certificates
  • Documentation: Comprehensive documentation of PKI infrastructure

Security Practices

  • CA Security: Protect CA private keys with HSM
  • Access Control: Restrict certificate issuance authority
  • Audit Logging: Log all certificate operations
  • Regular Reviews: Periodic security assessments
  • Disaster Recovery: Backup and recovery procedures

Emerging Trends in PKI

Automated Certificate Management

ACME protocol (used by Let's Encrypt) enables automated certificate lifecycle:

  • Automated domain validation
  • Programmatic certificate issuance
  • Automatic renewal before expiration
  • Integration with DevOps pipelines

Short-Lived Certificates

Trend toward shorter certificate validity periods:

  • Current Standard: 398 days maximum for public SSL certificates
  • Future Direction: Moving toward 90-day or shorter validity
  • Benefits: Reduced impact of compromise, forced regular rotation
  • Challenge: Requires automation for practical management

Certificate Transparency

Public logs of all issued certificates for accountability:

  • Detect mis-issued certificates
  • Monitor certificate issuance for domains
  • Improve CA accountability
  • Required for Extended Validation certificates

Key Takeaways

  • PKI provides framework for certificate management and trust establishment
  • Public CAs offer broad trust; private CAs provide control and cost savings
  • Certificate validation levels (DV, OV, EV) offer different trust assurances
  • Certificate revocation uses CRL or OCSP to invalidate compromised certificates
  • Selecting appropriate CA depends on use case, budget, and trust requirements
  • Automation and short-lived certificates are emerging best practices
