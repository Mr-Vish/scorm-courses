Web Accessibility and ARIA

Web Accessibility Fundamentals

Web accessibility ensures that websites, tools, and technologies are designed and developed so that people with disabilities can use them. This includes the ability to perceive, understand, navigate, and interact with the web.

Common Web Accessibility Barriers

BarrierImpactAffected Users
Missing alt text on imagesImages are invisible to screen readersBlind and low-vision users
Poor color contrastText is difficult or impossible to readLow-vision users, color blind users
Keyboard-inaccessible controlsCannot navigate or interact with siteMotor disability users, keyboard-only users
Missing form labelsCannot understand what input is requiredScreen reader users, cognitive disability users
Auto-playing mediaInterferes with screen reader audioBlind users, cognitive disability users
Time-limited contentInsufficient time to read or interactCognitive disability users, motor disability users

AI-Powered Accessibility Auditing

AI can automatically scan websites and identify accessibility issues:

from openai import OpenAI
from bs4 import BeautifulSoup
import requests

client = OpenAI()

def audit_webpage_accessibility(url: str) -> dict:
    '''Audit a webpage for accessibility issues.'''
    response = requests.get(url)
    soup = BeautifulSoup(response.content, 'html.parser')
    
    issues = []
    
    # Check for images without alt text
    images = soup.find_all('img')
    for img in images:
        if not img.get('alt'):
            issues.append({
                "severity": "Critical",
                "issue": "Missing alt text",
                "element": str(img)[:100],
                "wcag": "1.1.1 Non-text Content"
            })
    
    # Check for form inputs without labels
    inputs = soup.find_all('input')
    for input_elem in inputs:
        input_id = input_elem.get('id')
        if input_id:
            label = soup.find('label', {'for': input_id})
            if not label and not input_elem.get('aria-label'):
                issues.append({
                    "severity": "Critical",
                    "issue": "Form input without label",
                    "element": str(input_elem)[:100],
                    "wcag": "3.3.2 Labels or Instructions"
                })
    
    return {
        "url": url,
        "total_issues": len(issues),
        "critical": len([i for i in issues if i['severity'] == 'Critical']),
        "issues": issues
    }

ARIA (Accessible Rich Internet Applications)

ARIA is a set of attributes that define ways to make web content and applications more accessible to people with disabilities. It helps bridge gaps in HTML semantics, especially for dynamic content and complex user interface controls.

Common ARIA Attributes

AttributePurposeExample
aria-labelProvides accessible name for element<button aria-label="Close dialog">X</button>
aria-labelledbyReferences element that labels current element<div aria-labelledby="dialog-title">
aria-describedbyReferences element that describes current element<input aria-describedby="password-requirements">
aria-hiddenHides element from assistive technologies<span aria-hidden="true">â˜…</span>
aria-liveAnnounces dynamic content changes<div aria-live="polite">Loading...</div>
aria-expandedIndicates if element is expanded or collapsed<button aria-expanded="false">Menu</button>

AI-Generated ARIA Labels

AI can analyze UI components and generate appropriate ARIA labels:

def generate_aria_labels(html_snippet: str, page_context: str) -> str:
    '''Generate appropriate ARIA labels for HTML elements.'''
    response = client.chat.completions.create(
        model="gpt-4o",
        messages=[{
            "role": "user",
            "content": (
                "Analyze this HTML snippet and add appropriate ARIA attributes "
                "for accessibility. Include aria-label, aria-labelledby, "
                "aria-describedby, and role attributes where needed.\n\n"
                f"Page context: {page_context}\n\n"
                f"HTML:\n{html_snippet}\n\n"
                "Return the improved HTML with ARIA attributes."
            )
        }]
    )
    return response.choices[0].message.content

Keyboard Navigation Testing

All interactive elements must be keyboard accessible. Key requirements:

  • Tab navigation: All interactive elements reachable via Tab key
  • Focus indicators: Visible outline showing which element has focus
  • Enter/Space activation: Buttons and links activate with keyboard
  • Escape key: Closes modals and dismisses popups
  • Arrow keys: Navigate within menus, tabs, and lists

Real-World Implementation

  • E-commerce: Ensure product pages, checkout flows are fully accessible
  • Government: Section 508 compliance for federal websites
  • Education: Learning management systems accessible to all students
  • Healthcare: Patient portals accessible to elderly and disabled users
  • Banking: Financial services accessible to all customers

Key Takeaways

  • Web accessibility removes barriers for people with disabilities
  • ARIA attributes enhance HTML semantics for assistive technologies
  • AI can automatically audit websites and identify accessibility issues
  • Color contrast, keyboard navigation, and semantic HTML are critical
  • Automated testing should be combined with manual testing by users with disabilities
