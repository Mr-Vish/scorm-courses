Introduction to WCAG 2.1

The Web Content Accessibility Guidelines (WCAG) 2.1 are the internationally recognized standard for web accessibility, published by the World Wide Web Consortium (W3C) Web Accessibility Initiative (WAI). WCAG 2.1, released in June 2018, builds upon WCAG 2.0 (2008) by adding 17 new success criteria addressing mobile accessibility, low vision, and cognitive disabilities.

WCAG 2.1 is technology-neutral, meaning it applies to all web content regardless of the underlying technologyâ€”HTML, PDF, mobile applications, or emerging platforms. The guidelines are organized around four fundamental principles known as POUR.

The Four POUR Principles

1. Perceivable

Definition: Information and user interface components must be presentable to users in ways they can perceive. Content cannot be invisible to all of a user's senses.

Key requirements:

  • Text Alternatives: Provide text alternatives for non-text content (images, audio, video) so it can be changed into other forms people need, such as large print, braille, speech, symbols, or simpler language
  • Time-based Media: Provide alternatives for time-based media (captions for videos, transcripts for audio)
  • Adaptable: Create content that can be presented in different ways without losing information or structure
  • Distinguishable: Make it easier for users to see and hear content, including separating foreground from background

AI relevance: AI-powered alt text generation, automatic captioning, and audio description directly address Perceivable requirements by creating text alternatives at scale.

2. Operable

Definition: User interface components and navigation must be operable. Users must be able to interact with all controls and interactive elements.

Key requirements:

  • Keyboard Accessible: Make all functionality available from a keyboard (no mouse required)
  • Enough Time: Provide users enough time to read and use content; avoid automatic timeouts
  • Seizures and Physical Reactions: Do not design content in a way that is known to cause seizures or physical reactions (e.g., flashing content)
  • Navigable: Provide ways to help users navigate, find content, and determine where they are
  • Input Modalities: Make it easier for users to operate functionality through various inputs beyond keyboard

AI relevance: AI can assist in identifying navigation issues, detecting flashing content, and suggesting improvements to interactive elements.

3. Understandable

Definition: Information and the operation of the user interface must be understandable. Users must be able to comprehend the information and how to use the interface.

Key requirements:

  • Readable: Make text content readable and understandable; identify the language of the page and parts
  • Predictable: Make web pages appear and operate in predictable ways; consistent navigation and identification
  • Input Assistance: Help users avoid and correct mistakes; provide error identification, labels, and suggestions

AI relevance: AI-powered text simplification, readability analysis, and plain language transformation support Understandable requirements, particularly for cognitive accessibility.

4. Robust

Definition: Content must be robust enough that it can be interpreted reliably by a wide variety of user agents, including assistive technologies.

Key requirements:

  • Compatible: Maximize compatibility with current and future user agents, including assistive technologies
  • Valid code: Use clean, standards-compliant markup that assistive technologies can parse
  • Name, Role, Value: Ensure all user interface components have programmatically determinable names, roles, and values

AI relevance: AI tools can validate code, detect missing ARIA attributes, and suggest semantic HTML improvements to enhance robustness.

WCAG Conformance Levels

WCAG 2.1 defines three levels of conformance, each building upon the previous level:

LevelDescriptionTypical Use CaseSuccess Criteria Count
Level A Minimum level of conformance. Addresses the most basic accessibility features. Failure to meet Level A creates significant barriers for users with disabilities. Absolute minimum for any website 30 criteria
Level AA Recommended level for most organizations. Addresses the most common barriers for users with disabilities. Includes all Level A criteria plus additional requirements. Industry standard; legal requirement in most jurisdictions 20 additional criteria (50 total)
Level AAA Highest level of conformance. Includes all Level A and AA criteria plus the most stringent requirements. Not recommended as a general policy for entire sites. Specialized content requiring maximum accessibility 28 additional criteria (78 total)

Industry standard: WCAG 2.1 Level AA is the widely accepted target for most organizations and is referenced in most accessibility laws and regulations worldwide.

Key Success Criteria for AI-Powered Accessibility

1.1.1 Non-text Content (Level A)

Requirement: All non-text content that is presented to the user has a text alternative that serves the equivalent purpose.

AI application: Vision-language models can automatically generate descriptive alt text for images, reducing manual effort by 90%+.

Example:

<img src="product.jpg" alt="Stainless steel coffee maker with glass carafe and programmable timer">

1.2.2 Captions (Prerecorded) (Level A)

Requirement: Captions are provided for all prerecorded audio content in synchronized media.

AI application: Speech-to-text models like Whisper can automatically transcribe and generate synchronized captions for video content.

1.2.5 Audio Description (Prerecorded) (Level AA)

Requirement: Audio description is provided for all prerecorded video content in synchronized media.

AI application: AI can analyze video frames, identify visual elements, and generate audio description scripts describing important visual information not conveyed through dialogue.

1.4.3 Contrast (Minimum) (Level AA)

Requirement: The visual presentation of text and images of text has a contrast ratio of at least 4.5:1 (or 3:1 for large text).

AI application: AI tools can automatically detect contrast violations and suggest color adjustments to meet requirements.

1.4.5 Images of Text (Level AA)

Requirement: If the technologies being used can achieve the visual presentation, text is used to convey information rather than images of text.

AI application: Optical Character Recognition (OCR) combined with AI can extract text from images and suggest replacing images of text with actual text.

2.4.6 Headings and Labels (Level AA)

Requirement: Headings and labels describe topic or purpose.

AI application: Natural language processing can analyze content and suggest appropriate heading structures and descriptive labels.

3.1.5 Reading Level (Level AAA)

Requirement: When text requires reading ability more advanced than lower secondary education level, supplemental content or an alternative version is available.

AI application: Large language models can automatically simplify complex text, generate plain language summaries, and assess readability levels.

Legal Requirements and Compliance

United States

Americans with Disabilities Act (ADA)

The ADA prohibits discrimination based on disability. While the ADA does not explicitly mention websites, courts have increasingly ruled that websites are "places of public accommodation" subject to ADA requirements. The Department of Justice has indicated that WCAG 2.1 Level AA is the appropriate standard.

Enforcement: Private lawsuits (Title III) have increased dramatically, with over 4,000 federal lawsuits filed in 2021 alone. Settlements often require WCAG 2.1 Level AA conformance and ongoing monitoring.

Section 508 of the Rehabilitation Act

Section 508 requires federal agencies and their contractors to make electronic and information technology accessible. The Section 508 standards were updated in 2017 to incorporate WCAG 2.0 Level AA (with some exceptions).

Scope: Applies to federal government websites, applications, and documents, as well as vendors selling to the federal government.

European Union

European Accessibility Act (EAA)

The EAA, effective June 2025, requires accessibility for a wide range of products and services, including websites, mobile applications, e-commerce, banking, and e-books. Member states must transpose the directive into national law.

Standard: References EN 301 549, which incorporates WCAG 2.1 Level AA.

Web Accessibility Directive

Requires public sector websites and mobile applications to meet WCAG 2.1 Level AA standards. Member states must conduct regular monitoring and publish accessibility statements.

United Kingdom

Equality Act 2010

Prohibits discrimination based on disability. Service providers must make "reasonable adjustments" to ensure services are accessible, including websites and digital services.

Public Sector Bodies Accessibility Regulations 2018

Requires public sector websites and mobile apps to meet WCAG 2.1 Level AA standards and publish accessibility statements.

Canada

Accessible Canada Act (ACA)

Federal legislation requiring federally regulated organizations to identify, remove, and prevent accessibility barriers. Regulations are being developed to specify technical standards.

Accessibility for Ontarians with Disabilities Act (AODA)

Provincial legislation in Ontario requiring organizations to meet accessibility standards, including WCAG 2.0 Level AA for websites (with WCAG 2.1 expected in future updates).

Australia

Disability Discrimination Act 1992

Prohibits discrimination based on disability. The Australian Human Rights Commission has issued guidance indicating that WCAG 2.1 Level AA is the appropriate standard for web accessibility.

Consequences of Non-Compliance

Legal Liability

  • Lawsuits: Organizations face increasing litigation risk, with thousands of accessibility lawsuits filed annually
  • Settlements: Typical settlements range from $10,000 to $100,000+ plus attorney fees and mandatory remediation
  • Consent decrees: Court-ordered agreements requiring ongoing compliance monitoring and reporting

Financial Impact

  • Remediation costs: Retroactive accessibility fixes are 10-100x more expensive than building accessibility from the start
  • Lost revenue: Inaccessible websites exclude customers with disabilities and their families
  • Regulatory fines: Some jurisdictions impose financial penalties for non-compliance

Reputational Damage

  • Negative publicity: Accessibility lawsuits generate media coverage and social media criticism
  • Brand perception: Accessibility failures signal lack of social responsibility and inclusive values
  • Customer trust: Users with disabilities and their allies may boycott inaccessible brands

Compliance Strategy

Proactive Approach

  • Adopt WCAG 2.1 Level AA as organizational standard
  • Integrate accessibility into design and development processes
  • Conduct regular accessibility audits and testing
  • Provide accessibility training for all team members
  • Establish accessibility governance and accountability
  • Leverage AI tools to accelerate remediation and maintain compliance

Documentation

  • Accessibility statements: Publish commitment to accessibility and conformance level
  • VPAT (Voluntary Product Accessibility Template): Document conformance for enterprise customers
  • Remediation roadmap: Demonstrate good faith effort to address known issues
  • Testing reports: Maintain records of accessibility testing and validation

Key Takeaways

  • WCAG 2.1 is the international standard for web accessibility, organized around four POUR principles: Perceivable, Operable, Understandable, and Robust
  • WCAG 2.1 Level AA is the industry standard and legal requirement in most jurisdictions
  • Key success criteria for AI-powered accessibility include alt text (1.1.1), captions (1.2.2), audio description (1.2.5), and contrast (1.4.3)
  • Accessibility is legally mandated in the US (ADA, Section 508), EU (EAA, Web Accessibility Directive), UK (Equality Act), Canada (ACA, AODA), and Australia (DDA)
  • Non-compliance results in legal liability, financial costs, and reputational damage
  • Organizations should adopt WCAG 2.1 Level AA as standard, integrate accessibility into workflows, and leverage AI tools for efficient compliance
  • Proactive accessibility is significantly less expensive than retroactive remediation
