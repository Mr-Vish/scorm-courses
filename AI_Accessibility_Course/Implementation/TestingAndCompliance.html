Testing and Compliance

Accessibility Testing Fundamentals

Effective accessibility testing combines automated tools, manual testing, and user testing with people who have disabilities. No single approach catches all issues—a comprehensive strategy is essential.

Testing Pyramid

Testing LevelCoverageEffortTools
Automated~30% of issuesLowaxe, WAVE, Lighthouse, AI scanners
Manual~50% of issuesMediumScreen readers, keyboard testing, color contrast
User testing~20% of issuesHighReal users with disabilities

Automated Testing with AI

AI-powered tools can detect many accessibility issues automatically:

from openai import OpenAI
from selenium import webdriver
from axe_selenium_python import Axe

client = OpenAI()

def comprehensive_accessibility_test(url: str) -> dict:
    '''Run comprehensive automated accessibility test.'''
    driver = webdriver.Chrome()
    driver.get(url)
    
    # Run axe-core automated tests
    axe = Axe(driver)
    axe.inject()
    results = axe.run()
    
    # Get page HTML for AI analysis
    page_html = driver.page_source
    
    # AI-powered semantic analysis
    ai_analysis = analyze_with_ai(page_html)
    
    driver.quit()
    
    return {
        "url": url,
        "automated_issues": results["violations"],
        "ai_insights": ai_analysis,
        "total_issues": len(results["violations"]) + len(ai_analysis["issues"])
    }

def analyze_with_ai(html: str) -> dict:
    '''Use AI to detect semantic and contextual issues.'''
    response = client.chat.completions.create(
        model="gpt-4o",
        response_format={"type": "json_object"},
        messages=[{
            "role": "user",
            "content": (
                "Analyze this HTML for accessibility issues that automated "
                "tools might miss:\n"
                "1. Alt text quality (is it descriptive and contextual?)\n"
                "2. Heading hierarchy (logical structure?)\n"
                "3. Link text (descriptive or generic 'click here'?)\n"
                "4. Form labels (clear and helpful?)\n"
                "5. Language clarity (appropriate reading level?)\n\n"
                f"HTML:\n{html[:5000]}\n\n"
                "Return JSON with issues array."
            )
        }]
    )
    
    return json.loads(response.choices[0].message.content)

Screen Reader Testing

Testing with screen readers reveals issues automated tools miss:

User Testing with People with Disabilities

The gold standard for accessibility testing is testing with real users:

  • Recruit diverse participants: Different disabilities, assistive technologies, experience levels
  • Provide compensation: Pay participants fairly for their time and expertise
  • Use realistic tasks: Test actual user workflows, not artificial scenarios
  • Observe and listen: Watch how they interact, ask about pain points
  • Iterate based on feedback: Fix issues and retest

Accessibility Conformance Report (ACR)

Document your accessibility conformance using VPAT (Voluntary Product Accessibility Template):

def generate_accessibility_report(test_results: dict) -> str:
    '''Generate WCAG conformance report.'''
    response = client.chat.completions.create(
        model="gpt-4o",
        messages=[{
            "role": "user",
            "content": (
                "Generate a WCAG 2.1 Level AA conformance report based on these "
                "test results. Include:\n"
                "1. Executive summary\n"
                "2. Conformance level achieved\n"
                "3. Issues by WCAG criterion\n"
                "4. Remediation recommendations\n"
                "5. Timeline for fixes\n\n"
                f"Test results:\n{json.dumps(test_results, indent=2)}"
            )
        }]
    )
    
    return response.choices[0].message.content

Continuous Monitoring

Accessibility is not a one-time project—implement ongoing monitoring:

  • Automated daily scans: Check for new accessibility issues
  • Pre-deployment checks: Block releases with critical issues
  • User feedback channels: Easy way for users to report issues
  • Quarterly audits: Comprehensive manual testing
  • Annual user testing: Test with people with disabilities

Remediation Prioritization

Not all issues are equal—prioritize based on impact and effort:

PriorityCriteriaExamplesTimeline
CriticalBlocks access to core functionalityKeyboard-inaccessible checkout, missing form labelsFix immediately
HighSignificant barrier, affects many usersMissing alt text on key images, poor color contrastFix within 1 week
MediumModerate barrier, workarounds existHeading hierarchy issues, generic link textFix within 1 month
LowMinor issue, minimal impactMissing lang attribute, minor ARIA improvementsFix in next sprint

Documentation and Training

Maintain accessibility documentation for your team:

  • Accessibility guidelines: Your organization's standards and best practices
  • Component library: Accessible, reusable UI components
  • Testing procedures: How to test for accessibility
  • Remediation guides: How to fix common issues
  • Contact information: Who to ask for accessibility help

Third-Party Content

Ensure vendors and partners meet accessibility requirements:

  • Contract requirements: Include WCAG conformance in vendor contracts
  • VPAT review: Request and review Voluntary Product Accessibility Templates
  • Testing: Test third-party tools and content before deployment
  • Remediation responsibility: Clarify who fixes accessibility issues

Key Takeaways

  • Combine automated, manual, and user testing for comprehensive coverage
  • Automated tools catch ~30% of issues—manual testing is essential
  • Test with real users who have disabilities for best results
  • WCAG 2.1 Level AA is the standard for most organizations
  • Implement continuous monitoring, not one-time testing
  • Prioritize issues based on impact and fix critical blockers immediately
