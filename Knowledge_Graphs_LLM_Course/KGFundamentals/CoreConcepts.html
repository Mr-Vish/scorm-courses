Knowledge Graph Fundamentals

What is a Knowledge Graph?

A knowledge graph is a structured representation of knowledge that captures entities, their attributes, and the relationships between them. Unlike traditional databases that store data in tables or documents, knowledge graphs organize information as an interconnected network of nodes and edges, enabling intuitive traversal and complex reasoning.

Core Components

ComponentDescriptionExample
Entity (Node) A distinct object, concept, or thing in the real world Person: "Marie Curie", Organization: "University of Paris"
Relationship (Edge) A connection or association between two entities "Marie Curie" WORKED_AT "University of Paris"
Property (Attribute) A characteristic or data value associated with an entity or relationship birth_year: 1867, nationality: "Polish-French"
Triple The fundamental unit: (subject, predicate, object) (Marie Curie, WON, Nobel Prize in Physics)

Knowledge Graph vs Traditional Databases

AspectRelational DatabaseKnowledge Graph
Data Model Tables with rows and columns Nodes and edges (graph structure)
Relationships Foreign keys and joins First-class citizens (edges)
Schema Rigid, predefined schema Flexible, evolving schema
Query Complexity Complex joins for multi-hop queries Natural graph traversal
Best Use Case Transactional data, structured records Connected data, semantic relationships

Real-World Applications

  • Search Engines: Google's Knowledge Graph powers rich search results with entity information and relationships
  • Recommendation Systems: Netflix and Amazon use knowledge graphs to understand user preferences and content relationships
  • Drug Discovery: Pharmaceutical companies map relationships between genes, proteins, diseases, and compounds
  • Fraud Detection: Financial institutions identify suspicious patterns through relationship analysis
  • Enterprise Knowledge Management: Organizations connect documents, people, projects, and expertise
  • Question Answering: Virtual assistants leverage knowledge graphs for factual responses

Knowledge Graph Standards

RDF (Resource Description Framework)

RDF is a W3C standard for representing information as triples. Each triple consists of subject-predicate-object:

<http://example.org/person/marie_curie> 
  <http://example.org/ontology/wonAward> 
  <http://example.org/award/nobel_physics_1903> .

<http://example.org/person/marie_curie> 
  <http://example.org/ontology/birthYear> 
  "1867"^^xsd:integer .

Property Graph Model

Used by databases like Neo4j, this model allows properties on both nodes and relationships:

// Cypher query language example
CREATE (marie:Person {
  name: "Marie Curie",
  birthYear: 1867,
  nationality: "Polish-French"
})

CREATE (nobel:Award {
  name: "Nobel Prize in Physics",
  year: 1903
})

CREATE (marie)-[:WON {
  sharedWith: "Pierre Curie, Henri Becquerel"
}]->(nobel)

Key Characteristics of Effective Knowledge Graphs

  • Semantic Richness: Entities and relationships have well-defined meanings
  • Interconnectedness: High degree of connections enabling multi-hop reasoning
  • Scalability: Ability to grow from thousands to billions of entities
  • Queryability: Support for complex graph traversal and pattern matching
  • Evolvability: Schema can adapt as new entity types and relationships emerge
  • Provenance: Track sources and confidence levels for information
