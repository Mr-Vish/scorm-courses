Agent Architecture and Setup

Bedrock Agent Components

  • Foundation Model: The LLM that powers reasoning (Claude, Titan, Llama)
  • Instructions: System prompt defining agent role and behavior
  • Action Groups: Sets of API actions the agent can perform via Lambda
  • Knowledge Bases: RAG-enabled document stores for grounding
  • Guardrails: Content safety and topic filtering

Creating an Agent

import boto3

bedrock_agent = boto3.client("bedrock-agent")

response = bedrock_agent.create_agent(
    agentName="CustomerServiceAgent",
    foundationModel="anthropic.claude-3-sonnet-20240229-v1:0",
    instruction="You are a customer service agent for Acme Corp. "
        "You can look up orders, process returns, and answer product questions. "
        "Always be polite and verify the customer's identity before sharing order details.",
    idleSessionTTLInSeconds=1800
)

Action Groups with OpenAPI Schema

Define action groups using OpenAPI specifications that map to Lambda functions:

# OpenAPI schema for order lookup
openapi: 3.0.0
paths:
  /orders/{orderId}:
    get:
      summary: Look up an order by ID
      operationId: getOrder
      parameters:
        - name: orderId
          in: path
          required: true
          schema:
            type: string
