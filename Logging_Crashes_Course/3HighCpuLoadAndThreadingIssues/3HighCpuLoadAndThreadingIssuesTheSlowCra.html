3. High CPU Load and Threading Issues (The "Slow Crash")

While not an instant crash, excessive logging can cause the application to become unresponsive, which is often considered a failure state.

  • Synchronous Logging: If logging is purely synchronous, high-volume logging forces main application threads to block while waiting for disk I/O. This leads to extreme latency and application timeouts or deadlocks under load.
  • Excessive Logging in Loops: Logging inside tight, frequently executed loops can saturate the CPU with string formatting, context gathering, and I/O operations. This excessive CPU usage starves other critical application threads, leading to thread pool exhaustion and application unresponsiveness.
