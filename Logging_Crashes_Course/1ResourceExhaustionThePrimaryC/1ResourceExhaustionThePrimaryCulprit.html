1. Resource Exhaustion: The Primary Culprit

The most frequent way logging leads to an application crash is by exhausting system resources.

A. Disk Space Depletion (Rolling File Appenders)

If your application uses a file appender (e.g., RollingFileAppender) and is running with a high logging level (like DEBUG or TRACE) under heavy load, it can generate enormous amounts of log data rapidly.

  • The Problem: If no log rotation or size/time capping is configured, or if the configured limits are too high, the log files will consume all available disk space on the server.
  • The Crash: Once the disk is 100% full, the JVM (Java Virtual Machine) and the Spring Boot application may crash or hang because they can no longer write new log entries or perform internal operations requiring temporary disk space.
  • Safeguard: Use size- and time-based rolling policies and ensure an appropriate maximum file size/count is set.
