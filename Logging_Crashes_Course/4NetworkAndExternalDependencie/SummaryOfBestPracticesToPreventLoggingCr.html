Summary of Best Practices to Prevent Logging Crashes

Scenario Risk Factor Prevention Strategy
Disk Space Rapid log file growth (DEBUG/TRACE) Implement rolling policies (size & time) and set a maximum file count/total size.
Memory (OOM) Log4j2 queue overflow Use Async Loggers with a defined RingBufferSize or Async Appenders with a limited queueSize and onOverflow="DISCARD".
Performance Synchronous I/O in high-load areas Utilize fully asynchronous logging (e.g., Log4j2 Async Loggers) exclusively.
External Service Remote log service being unavailable Configure timeouts and fail-safes (e.g., non-blocking or dropping events).
Startup Configuration errors Validate logging configuration files and ensure correct dependencies are present.
