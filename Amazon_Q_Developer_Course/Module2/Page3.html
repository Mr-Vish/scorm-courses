Module 2: Enterprise Customization & Administration

Enterprise Customization

A key differentiator of Amazon Q Developer for enterprise customers is the ability to Customize its suggestions. While the base model is trained on public code, your organization likely has proprietary libraries, internal APIs, and specific coding standards that are not public knowledge.

How Customization Works

Using Retrieval Augmented Generation (RAG), Amazon Q can securely index your private code repositories (e.g., in AWS CodeCommit, GitHub, or GitLab). When a developer writes code that references internal libraries, Amazon Q retrieves relevant snippets from your indexed codebase and uses them as context.

This results in suggestions that:

  • Use internal APIs correctly (e.g., calling CompanyAuth.validate() instead of a generic auth function).
  • Adhere to team coding styles and conventions.
  • Accelerate onboarding for new hires by exposing them to existing patterns.

Administrative Controls

Enterprise administrators have granular control over how Amazon Q is used across the organization via the AWS Console.

Policy Management

  • Block Suggestions with Public References: Ensure that no code suggestions match public open-source code unless the license is explicitly allowed. This mitigates IP risk.
  • Limit Code Scanning: Define which repositories or projects are scanned for security vulnerabilities.
  • Access Control: Grant or revoke access to Amazon Q features based on IAM roles and groups.

Monitoring and Reporting

Administrators can view comprehensive dashboards showing:

  • Adoption: Number of active users and suggestion acceptance rates.
  • Productivity Impact: Metrics on lines of code accepted and time saved (estimated).
  • Security Findings: Aggregate view of vulnerabilities detected across the organization's codebases.

Summary

For large organizations, generic AI assistance is not enough. Amazon Q Developer provides the customization and control necessary to integrate AI into enterprise workflows safely and effectively. By tailoring the AI to your specific codebase and enforcing strict governance policies, you can maximize the benefits of generative AI while minimizing risk.
