Introduction to Setup and Configuration

Proper configuration of Amazon Q Developer is essential for maximizing productivity while maintaining security and compliance. The setup process varies depending on your development environment, organizational requirements, and licensing tier. This lesson provides comprehensive guidance for installing, configuring, and optimizing Amazon Q Developer across different scenarios.

Installation Prerequisites

System Requirements

Before installing Amazon Q Developer, ensure your system meets these requirements:

Hardware Requirements

  • Processor: Modern multi-core processor (Intel i5/AMD Ryzen 5 or better recommended)
  • Memory: Minimum 8GB RAM (16GB recommended for large projects)
  • Storage: 500MB free disk space for extensions and cache
  • Network: Stable internet connection for real-time suggestions

Software Requirements

  • Operating System: Windows 10/11, macOS 10.15+, or Linux (Ubuntu 18.04+, RHEL 7+)
  • IDE Version: Latest stable version of your chosen IDE
  • Browser: Modern browser for authentication (Chrome, Firefox, Safari, Edge)

Account Requirements

You need one of the following to use Amazon Q Developer:

  • AWS Builder ID: Free personal account for individual developers
  • IAM Identity Center: For enterprise SSO integration
  • AWS Account: For Pro tier with billing to AWS account

Installation for Visual Studio Code

Step-by-Step Installation

1. Install the Extension

There are multiple methods to install the Amazon Q extension:

Method A: VS Code Marketplace

  • Open VS Code
  • Click the Extensions icon in the sidebar (or press Ctrl+Shift+X / Cmd+Shift+X)
  • Search for "Amazon Q"
  • Click "Install" on the official AWS extension

Method B: Command Line

code --install-extension amazonwebservices.amazon-q-vscode

Method C: Manual Installation

  • Download the .vsix file from the VS Code Marketplace website
  • In VS Code, go to Extensions → More Actions (···) → Install from VSIX
  • Select the downloaded file

2. Initial Authentication

After installation, authenticate to activate features:

  • Click the Amazon Q icon in the sidebar
  • Select "Sign in to get started"
  • Choose authentication method:
    • AWS Builder ID: For free tier individual use
    • IAM Identity Center: For enterprise SSO
    • AWS Account: For Pro tier billing
  • Complete authentication in your browser
  • Return to VS Code to confirm connection

3. Verify Installation

Confirm Amazon Q is working correctly:

  • Open a code file in a supported language
  • Type a comment describing a function (e.g., "// function to sort array of numbers")
  • Press Enter and wait for inline suggestions to appear
  • Open the chat panel and ask a question

Installation for JetBrains IDEs

Supported JetBrains Products

Amazon Q supports all major JetBrains IDEs:

  • IntelliJ IDEA (Community and Ultimate)
  • PyCharm (Community and Professional)
  • WebStorm
  • PhpStorm
  • GoLand
  • RubyMine
  • CLion
  • Rider

Installation Process

1. Install the Plugin

Via JetBrains Marketplace:

  • Open your JetBrains IDE
  • Go to File → Settings (Windows/Linux) or Preferences (macOS)
  • Navigate to Plugins
  • Click the Marketplace tab
  • Search for "Amazon Q"
  • Click Install and restart the IDE

Via Command Line:

# For IntelliJ IDEA idea installPlugins com.aws.amazonq

2. Configure Authentication

  • After restart, click the Amazon Q icon in the toolbar
  • Select authentication method
  • Complete browser-based authentication
  • Verify connection in the Amazon Q tool window

Command Line Interface Setup

Installation

The Amazon Q CLI provides terminal-based assistance:

macOS/Linux:

curl -fsSL https://q.aws.dev/install.sh | bash

Windows (PowerShell):

iwr -useb https://q.aws.dev/install.ps1 | iex

CLI Configuration

After installation, configure the CLI:

q configure

This command guides you through:

  • Authentication method selection
  • Default region configuration
  • Output format preferences
  • Proxy settings (if required)

Enterprise Configuration

Identity and Access Management

IAM Identity Center Integration

Enterprise organizations typically use IAM Identity Center (formerly AWS SSO) for centralized authentication:

Administrator Setup:

  • Create an IAM Identity Center instance in your AWS Organization
  • Configure identity source (Active Directory, Okta, Azure AD, etc.)
  • Create permission sets for Amazon Q Developer access
  • Assign users or groups to permission sets
  • Distribute start URL to developers

Developer Connection:

  • In IDE, select "Sign in with IAM Identity Center"
  • Enter organization's start URL
  • Authenticate through corporate identity provider
  • Approve device authorization

Permission Management

Control feature access through IAM policies:

{ "Version": "2012-10-17", "Statement": [ { "Effect": "Allow", "Action": [ "codewhisperer:GenerateRecommendations", "codewhisperer:GetCodeAnalysis" ], "Resource": "*" } ] }

Organizational Policies

Content Filtering

Administrators can configure what types of suggestions are allowed:

  • Language Restrictions: Limit to approved programming languages
  • Framework Restrictions: Block suggestions for deprecated frameworks
  • License Filtering: Automatically reject code matching certain licenses
  • Sensitive Pattern Blocking: Prevent transmission of regex patterns matching secrets

Usage Monitoring

Track adoption and usage across the organization:

  • Acceptance Rate: Percentage of suggestions accepted by developers
  • Feature Utilization: Which features (completion, chat, scan) are most used
  • Language Distribution: Which programming languages are being assisted
  • Security Findings: Aggregate vulnerability detection statistics

Network and Proxy Configuration

Corporate Proxy Settings

Many enterprises require proxy configuration for internet access:

VS Code Proxy Configuration

Edit settings.json:

{ "http.proxy": "http://proxy.company.com:8080", "http.proxyStrictSSL": true, "http.proxyAuthorization": "Basic base64encodedcredentials" }

JetBrains Proxy Configuration

  • Go to Settings → Appearance & Behavior → System Settings → HTTP Proxy
  • Select "Manual proxy configuration"
  • Enter proxy host and port
  • Configure authentication if required

Firewall Requirements

Ensure these endpoints are accessible:

  • codewhisperer.*.amazonaws.com: Core service endpoints
  • *.q.aws.dev: Amazon Q specific services
  • sso.*.amazonaws.com: Authentication services
  • *.cloudfront.net: Content delivery for models and updates

Optimization and Best Practices

Performance Tuning

For Large Codebases

  • Exclude Directories: Prevent scanning of node_modules, build outputs, and vendor directories
  • File Size Limits: Configure maximum file size for analysis (default 1MB)
  • Indexing Optimization: Allow IDE indexing to complete before heavy Amazon Q usage

For Remote Development

  • Latency Considerations: Increase suggestion delay for high-latency connections
  • Bandwidth Management: Disable automatic security scanning on metered connections
  • Local Caching: Enable aggressive caching of frequently used suggestions

Workflow Integration

Version Control Integration

  • Configure pre-commit hooks to run security scans
  • Add .gitignore entries for Amazon Q cache directories
  • Document AI-generated code in commit messages

CI/CD Pipeline Integration

  • Integrate security scanning into build pipelines
  • Fail builds on critical vulnerabilities
  • Generate compliance reports for audit trails

Troubleshooting Common Issues

Authentication Problems

Issue Possible Cause Solution
Login fails repeatedly Expired credentials or network issues Clear cached credentials and re-authenticate
SSO not working Incorrect start URL or permissions Verify start URL with administrator; check IAM permissions
Browser doesn't open Default browser not configured Manually copy authentication URL to browser

Suggestion Issues

  • No suggestions appearing: Check internet connection, verify authentication, ensure file type is supported
  • Slow suggestions: Check network latency, reduce suggestion delay, verify proxy configuration
  • Irrelevant suggestions: Provide more context through comments, ensure file is properly formatted

Performance Issues

  • IDE slowdown: Exclude large directories, increase IDE memory allocation, disable unused features
  • High CPU usage: Reduce scanning frequency, limit concurrent operations
  • Network congestion: Configure bandwidth limits, schedule scans during off-peak hours

Key Takeaways

  • Amazon Q Developer supports multiple IDEs with consistent functionality across platforms
  • Installation is straightforward through marketplace/plugin systems for each IDE
  • Authentication options include AWS Builder ID for individuals and IAM Identity Center for enterprises
  • Extensive configuration options allow customization for individual preferences and organizational policies
  • Enterprise deployments require careful planning for identity management, network access, and policy enforcement
  • Proper configuration of proxies and firewalls is essential for corporate environments
  • Performance optimization is important for large codebases and remote development scenarios
  • Integration with existing development workflows enhances productivity and maintains security
