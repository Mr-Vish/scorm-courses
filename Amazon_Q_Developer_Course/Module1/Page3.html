Module 1: Setup & Configuration

Setup & Configuration Overview

Before diving into coding, getting Amazon Q Developer properly configured is essential. The setup process varies slightly depending on whether you are using the Individual Tier (free) or the Professional Tier (enterprise), but the core steps remain consistent: installing the IDE extension and authenticating with AWS.

Authentication Methods

Amazon Q Developer supports two primary authentication mechanisms:

  • AWS Builder ID (Individual): A personal profile for developers. It's free, tied to your email, and independent of any specific AWS account or company. Best for individual developers exploring the tool.
  • IAM Identity Center (Professional): Enterprise-grade authentication managed by your organization's IT department. It integrates with your existing corporate directory (like Active Directory or Okta) and allows centralized management of access policies. This method unlocks the advanced enterprise features of the Pro tier.

Detailed Explanation: IDE Integration

Supported Environments

Amazon Q is available across a wide range of development environments:

  • Visual Studio Code: Install the "AWS Toolkit" extension.
  • JetBrains IDEs (IntelliJ IDEA, PyCharm, WebStorm, etc.): Install the "AWS Toolkit" plugin.
  • Visual Studio 2022: Install the "AWS Toolkit for Visual Studio 2022".
  • AWS Cloud9: Pre-installed in AWS's cloud-based IDE.
  • Command Line Interface (CLI): Integrated via "Amazon Q for command line" (formerly Fig).
  • macOS Terminal: Through the Amazon Q desktop app.

Step-by-Step Configuration (VS Code Example)

  1. Open VS Code and navigate to the Extensions marketplace.
  2. Search for "AWS Toolkit" and install it.
  3. Click on the AWS icon in the sidebar.
  4. Under "Amazon Q + CodeWhisperer," select "Start".
  5. Choose "Use for Free with AWS Builder ID" or "Use with IAM Identity Center" based on your subscription.
  6. Follow the browser prompts to authenticate and authorize the extension.

Once authenticated, you will see a status indicator showing that Amazon Q is active. You can then configure settings such as "Share Code for Service Improvement" (opt-out available for Pro users) and reference tracking behavior.

Enterprise Administration

For organizations, setup involves more than just individual IDE plugins. Administrators use the AWS Management Console to:

  • Enable Amazon Q: Activate the service in the desired AWS Region.
  • Manage Users: Assign licenses to users via IAM Identity Center.
  • Define Policies: Control which suggestions are allowed (e.g., block suggestions with references to code under specific licenses).
  • Monitor Usage: Track adoption metrics and security scan findings through a dashboard.

Practical Application: Customization

Enterprise administrators can also configure Customizations. This advanced feature allows Amazon Q to be aware of your organization's private libraries, APIs, and coding standards. By pointing Amazon Q to your private repositories (e.g., in CodeCommit or GitHub), it can learn your internal patterns and suggest code that aligns with your company's specific practices, significantly boosting developer onboarding speed.

Summary

Proper setup is the foundation of a productive experience with Amazon Q Developer. Whether you are an individual using AWS Builder ID or part of a large enterprise using IAM Identity Center, the integration is seamless across major IDEs. Understanding the administrative controls ensures that the tool is used securely and effectively at scale.
