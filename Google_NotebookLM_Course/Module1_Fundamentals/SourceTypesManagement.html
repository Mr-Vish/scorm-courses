Source Types and Management

Supported Source Types

NotebookLM supports a diverse range of source formats, each with specific capabilities and limitations. Understanding these differences helps you optimize your research workflow.

Source Type Integration Method Key Features Limitations
Google Docs Direct connection via Google Drive Live sync, automatic updates, collaborative editing Requires Google account permissions
PDF Files File upload Preserves formatting, supports scanned documents (OCR) 500K word limit per file, static content
Web URLs Paste link Captures public web content, articles, blog posts Snapshot at upload time, no dynamic updates
Text Files (.txt) File upload Simple, lightweight, fast processing No formatting preservation
Markdown Files (.md) File upload Structured content, code-friendly Formatting may be simplified
Google Slides Direct connection via Google Drive Extracts text and speaker notes Images and charts not analyzed
YouTube Videos Paste video URL Automatic transcript extraction Requires available captions/transcripts
Audio Files File upload (MP3, WAV) Automatic transcription Transcription accuracy varies with audio quality

Source Size and Quantity Limits

Understanding NotebookLM's capacity constraints helps you plan your research projects effectively:

  • Maximum Sources per Notebook: 50 sources
  • Maximum Words per Source: 500,000 words (approximately 1,000 pages)
  • Total Notebook Capacity: Up to 25 million words across all sources
  • File Size Limits: Varies by type, typically up to 500MB for PDFs

Adding Sources to Your Notebook

Method 1: Google Drive Integration

1. Click "Add Source" or the "+" button in your notebook 2. Select "Google Drive" 3. Navigate to your document (Docs, Slides, or Sheets) 4. Click "Select" to add the source 5. NotebookLM will process and index the document Note: Google Drive sources maintain a live connection and can be refreshed to reflect updates.

Method 2: File Upload

1. Click "Add Source" or the "+" button 2. Select "Upload" or drag and drop files 3. Choose PDF, TXT, MD, or audio files from your computer 4. Wait for processing to complete (typically 10-60 seconds) 5. The source appears in your sources list Tip: You can upload multiple files simultaneously by selecting them together.

Method 3: Web URL

1. Click "Add Source" 2. Select "Website URL" 3. Paste the complete URL of the web page 4. Click "Add" or press Enter 5. NotebookLM fetches and processes the content Important: Only publicly accessible URLs work. Password-protected or paywalled content cannot be added.

Method 4: YouTube Video

1. Click "Add Source" 2. Select "YouTube URL" 3. Paste the YouTube video link 4. NotebookLM extracts available transcripts/captions 5. The transcript becomes searchable and analyzable Note: Videos without captions or transcripts cannot be processed.

Source Management Best Practices

Organizing Sources

While NotebookLM doesn't have folder structures, you can organize sources effectively:

  • Naming Conventions: Rename sources with descriptive titles (e.g., "Smith2023_ClimateStudy" instead of "document.pdf")
  • Chronological Ordering: Add sources in logical sequence for your research flow
  • Thematic Grouping: Create separate notebooks for distinct topics or projects
  • Version Control: Include version numbers or dates in source names when tracking document revisions

Source Selection Strategy

Not all documents are equally valuable for NotebookLM analysis. Prioritize sources that:

  • Contain Substantive Text: Documents with meaningful content, not just images or tables
  • Are Well-Structured: Clear headings, paragraphs, and logical organization improve AI understanding
  • Complement Each Other: Sources that provide different perspectives or cover related aspects
  • Are Authoritative: Reliable, credible sources produce more trustworthy insights

Updating and Refreshing Sources

Google Drive Sources

Documents connected from Google Drive can be refreshed to reflect changes:

1. Locate the source in your notebook's source list 2. Click the refresh icon next to the source name 3. NotebookLM re-processes the updated content 4. Previous chat history remains, but new queries reflect updated content Use Case: Collaborative documents that team members are actively editing.

Static Sources (PDFs, URLs)

Uploaded files and web URLs are static snapshots. To update:

1. Remove the old version of the source 2. Upload or add the new version 3. Previous conversations referencing the old source remain but may become outdated Best Practice: Include version numbers in source names to track which version conversations reference.

Removing Sources

When a source is no longer needed or was added by mistake:

Source Processing and Indexing

Understanding what happens during source processing helps you anticipate capabilities and limitations:

Text Extraction

  • Plain Text: Directly indexed and searchable
  • PDFs: Text extracted via OCR if needed; formatting preserved where possible
  • Web Pages: Main content extracted; ads and navigation typically excluded
  • Presentations: Slide text and speaker notes extracted; images described if text is present

Semantic Indexing

NotebookLM creates semantic embeddings that capture meaning, not just keywords. This enables:

  • Conceptual searches that find relevant passages even without exact keyword matches
  • Cross-source connections based on related ideas
  • Context-aware responses that understand nuance and implication

Working with Large Document Collections

When working with many sources, consider these strategies:

Selective Source Inclusion

Instead of uploading everything, curate sources strategically:

  • Start with 5-10 core sources
  • Add supplementary sources as specific questions arise
  • Remove sources that prove less relevant

Multiple Notebooks Approach

For complex projects, create multiple notebooks:

  • Primary Research Notebook: Core sources and main analysis
  • Background Reading Notebook: Contextual and foundational materials
  • Methodology Notebook: Research methods and technical documentation
  • Synthesis Notebook: Generated notes and insights from other notebooks

Source Quality and AI Performance

The quality of your sources directly impacts NotebookLM's effectiveness:

Source Quality Factor Impact on AI Performance Recommendation
Text Clarity Clear text improves comprehension and citation accuracy Use high-quality PDFs; avoid heavily scanned or low-resolution documents
Structure Well-organized documents enable better section-specific queries Prefer sources with clear headings, sections, and logical flow
Relevance Irrelevant sources dilute focus and may introduce noise Be selective; quality over quantity
Consistency Contradictory sources require explicit handling in prompts Acknowledge conflicts; ask NotebookLM to compare perspectives
