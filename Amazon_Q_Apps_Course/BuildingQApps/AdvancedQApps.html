Advanced Q Apps and Sharing

Multi-Card App Design

Complex Q Apps chain multiple cards together for multi-step workflows:

  • Input Card: Collects user data (text, file, selection)
  • Query Card: Sends input to Q Business with specific instructions
  • Plugin Card: Calls an external API or Q Business plugin
  • Output Card: Displays formatted results

Example: RFP Response Generator

  1. Input Card 1: Upload RFP document (file input)
  2. Input Card 2: Select product line (dropdown)
  3. Query Card 1: Extract key requirements from RFP
  4. Query Card 2: Match requirements to product capabilities (uses enterprise data)
  5. Output Card: Formatted RFP response draft

Publishing and Sharing

Share your Q Apps with your organization:

  • Personal: Only you can access the app
  • Shared: Published to your organization's Q App Library
  • Admin-curated: Featured apps promoted by administrators

Data Source Integration

Q Apps access the same data sources configured in Q Business:

  • Amazon S3 buckets (documents, spreadsheets)
  • SharePoint sites and document libraries
  • Confluence spaces and pages
  • Salesforce knowledge articles
  • ServiceNow knowledge base

Governance and Controls

  • Administrators control who can create and share Q Apps
  • Apps respect existing Q Business access controls and guardrails
  • Usage analytics show which apps are most popular
  • Version history tracks changes to published apps
