Designing Accessible Apps with Cards

The Card Metaphor and Screen Readers

Amazon Q Apps uses a system of "cards" to organize information and actions. For users of screen readers, this modular approach is highly beneficial because it creates distinct landmarks within the application. However, as a builder, you must ensure that these cards are configured in a logical sequence to maintain a clear narrative flow.

Optimizing Different Card Types for Accessibility

Card Type Accessibility Best Practices
Input Card Provide clear, concise labels for all input fields. Use the "Instruction" field to give examples of valid input, which helps users with cognitive disabilities.
Query Card Ensure the query logic is focused. Broad, unfocused queries can result in "information overload," which is a significant barrier for many users.
Output Card Configure the output to use Markdown headers and lists. This ensures that screen reader users can navigate the generated content efficiently.

Prompt Engineering for Inclusive Outputs

The "Query" and "Content Creator" cards in Amazon Q Apps rely on prompts to generate their results. "Prompt Engineering" is the most effective tool a builder has for ensuring the AI's output is accessible.

The "System Prompt" Strategy

When configuring an output card, you can include specific instructions in the prompt to enforce accessibility standards. Examples of effective accessibility-focused instructions include:

  • "Please provide the output in plain English, avoiding complex metaphors or jargon."
  • "Structure the response using clear headings (H1, H2) and bullet points."
  • "If the data includes numbers, present them in a simple table with descriptive headers."
  • "Summarize long paragraphs into 2-3 key sentences to improve readability."

Managing User Interaction

Accessibility also involves how users interact with the app's components. In Amazon Q Apps, you should consider:

  • Selection Cards: Use these instead of free-text input when there are limited valid options. This reduces the motor and cognitive effort required to use the app.
  • Conditional Logic: While Q Apps handles much of the complexity, ensure that the flow between cards is intuitive. Avoid "loops" or confusing jumps that could disorient a user.
  • Feedback Mechanisms: If a card fails to produce a result (e.g., no data found), ensure the error message is helpful and provides a clear "next step" for the user.

Visual Design and Layout

While the Q Apps framework provides a consistent UI, builders can often choose how to arrange cards. A vertical, single-column layout is generally the most accessible, as it provides a predictable reading order for both sighted users and those using assistive technology. Avoid complex horizontal arrangements that might require excessive scrolling or "zooming" on mobile devices.

Key Takeaways

  • Modular card-based design provides predictable landmarks for screen reader users.
  • Prompt engineering is essential for instructing the AI to generate accessible content.
  • Selection cards and structured layouts reduce the physical and cognitive effort for users.
