Personalization and User Profiles

User Profile Schema

{
    "user_id": "user_123",
    "name": "Alice",
    "preferences": {
        "communication_style": "technical",
        "response_length": "concise",
        "language": "en"
    },
    "facts": [
        "Works on Spring Boot microservices",
        "Prefers PostgreSQL over MySQL",
        "Uses IntelliJ IDEA"
    ],
    "conversation_count": 47,
    "last_active": "2025-01-15"
}

Dynamic Personalization

  • Extract facts: After each conversation, extract new user facts using an LLM
  • Update preferences: Track implicit preferences from user behavior
  • Inject context: Include relevant user facts in the system prompt
  • Adapt tone: Adjust formality and detail level based on user profile

Privacy Considerations

PrincipleImplementation
ConsentAsk users before storing personal information
MinimizationOnly store facts necessary for better service
DeletionProvide "forget me" functionality
TransparencyLet users view and edit their stored profile
SecurityEncrypt stored profiles, restrict access

Combining Memory Types

Production chatbots typically combine multiple memory types: buffer memory for the current session, summary memory for long conversations, and long-term memory for user preferences and facts that persist across sessions.
