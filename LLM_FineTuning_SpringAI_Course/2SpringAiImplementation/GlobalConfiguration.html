Global Configuration for Fine-Tuned Models

For production environments, you typically want to configure your fine-tuned model ID globally using application.properties or application.yml. This allows you to change the model version across the entire app without modifying code.

Spring AI Properties

Configure your custom model path or ID in your configuration file:

application.yml:

spring:
  ai:
    openai:
      chat:
        options:
          model: ft:gpt-4o-mini:your-org:specialized-assistant-v2
          temperature: 0.1

Local Models with Ollama:

If you are running a fine-tuned model locally, you would point Spring AI to your local Ollama instance and the name of the model you created via a Modelfile:

spring:
  ai:
    ollama:
      chat:
        options:
          model: my-custom-llama3-ft

Why Global Configuration?

  • Environment Separation: Use a smaller base model in dev and your high-quality fine-tuned model in prod.
  • Version Management: Quickly roll back to a previous model version by simply changing a property value.
  • Security: Keep model IDs and API keys out of your source code.
