WARN vs. INFO: Distinguishing the Difference

The core difference lies in the expectation and consequence of the event being logged:

Feature WARN (Warning) INFO (Information)
Expectation Something unexpected happened, or a condition was met that might lead to a problem. Something expected happened, indicating normal progress or a key successful action.
Action Requires attention for possible investigation or future mitigation. Provides context or confirms success; typically requires no immediate action.
Example Database connection timed out but successfully retried. A configuration value is deprecated. Service started successfully. User logged in. A large batch process completed.
Tone Cautionary Declarative

In short, INFO logs are good news or neutral progress updates, while WARN logs are "heads-up" messages about something slightly off-kilter.
