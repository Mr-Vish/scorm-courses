Key Concepts in Spring AI

Core Components

To build a basic AI application with Spring AI, you need to understand three core components:

Concept Description Spring AI Class
Model The Large Language Model (LLM) itself (e.g., GPT-4, Gemini). ChatModel (or ChatClient)
Prompt The input or instructions you send to the model. Prompt
Completion The generated output or response received from the model. ChatResponse
ChatClient A high-level, fluent API that wraps the ChatModel to simplify sending prompts and retrieving content. This is what you'll use most often. ChatClient

Understanding the Model

Definition: The Model represents the Large Language Model that processes your requests and generates responses. Spring AI supports multiple providers including:

  • OpenAI (GPT-3.5, GPT-4, GPT-4o)
  • Google (Gemini)
  • Azure OpenAI
  • Anthropic (Claude)
  • Ollama (local models)

Working with Prompts

Definition: A Prompt is the input you provide to the AI model. It contains the instructions, context, and questions you want the model to process.

Prompts can be:

  • Simple text: Direct questions or instructions
  • Templated: Using placeholders for dynamic content
  • System prompts: Setting the behavior and personality of the AI
  • User prompts: The actual user input or query

Understanding Completions

Definition: A Completion (or ChatResponse) is the output generated by the AI model in response to your prompt. It contains:

  • The generated text content
  • Metadata about the response (tokens used, model version, etc.)
  • Multiple choice responses (if requested)

The ChatClient API

The ChatClient is the primary interface you'll use in Spring AI. It provides a fluent, builder-style API for interacting with AI models.

ChatClient simplifies the process of sending prompts and receiving responses, handling all the complexity of provider-specific APIs behind the scenes.

Key Features of ChatClient

  • Fluent API: Chain method calls for readable code
  • Auto-configuration: Spring Boot automatically configures it based on dependencies
  • Provider-agnostic: Same code works with different AI providers
  • Type-safe: Compile-time checking of your AI interactions

Spring AI Architecture

Spring AI follows the familiar Spring architecture patterns:

  • Dependency Injection: ChatClient is injected into your beans
  • Auto-configuration: Minimal setup required, works out of the box
  • Starter Dependencies: Simple Maven/Gradle dependencies to get started
  • Configuration Properties: API keys and settings in application.properties
