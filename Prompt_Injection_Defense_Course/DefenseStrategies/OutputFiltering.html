Output Filtering and Monitoring

Output Validation

  • Content classifiers: Run a second, smaller model to classify outputs for policy violations
  • Regex filters: Block outputs containing system prompt fragments, PII patterns, or known attack responses
  • Structured output enforcement: Force JSON schema compliance to prevent free-text injection responses

Monitoring and Alerting

SignalWhat to MonitorAction
Anomalous inputsKeywords like "ignore", "system prompt", "jailbreak"Flag for review, rate limit
Output anomaliesSudden topic shifts, system prompt leakageBlock response, alert team
Tool misuseUnexpected API calls or database queriesRequire human approval
Rate patternsRapid sequential probing attemptsTemporary ban, CAPTCHA

Architectural Defenses

  • Least privilege: Only give the LLM access to tools it absolutely needs
  • Human-in-the-loop: Require approval for high-impact actions
  • Separate trust domains: Don't mix user data and system instructions in the same context
  • Sandboxing: Run LLM-generated code in isolated environments
