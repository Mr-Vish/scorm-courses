Understanding Bedrock Guardrails

What Are Bedrock Guardrails?

Amazon Bedrock Guardrails provide configurable safeguards for your GenAI applications. They filter both input (user queries) and output (model responses) to enforce your organization's content policies.

Guardrail Components

  • Content Filters: Block harmful content across categories (hate, violence, sexual, misconduct, prompt attacks)
  • Denied Topics: Define custom topics that the model should refuse to discuss
  • Word Filters: Block specific words, phrases, or profanity
  • Sensitive Information Filters: Detect and redact PII (SSN, email, phone, credit card)
  • Contextual Grounding Check: Verify responses are grounded in provided context

Creating a Guardrail

import boto3

bedrock = boto3.client("bedrock")

response = bedrock.create_guardrail(
    name="CustomerServiceGuardrail",
    description="Guardrail for customer service chatbot",
    contentPolicyConfig={
        "filtersConfig": [
            {"type": "HATE", "inputStrength": "HIGH", "outputStrength": "HIGH"},
            {"type": "VIOLENCE", "inputStrength": "HIGH", "outputStrength": "HIGH"},
            {"type": "SEXUAL", "inputStrength": "HIGH", "outputStrength": "HIGH"},
            {"type": "MISCONDUCT", "inputStrength": "HIGH", "outputStrength": "HIGH"},
            {"type": "PROMPT_ATTACK", "inputStrength": "HIGH", "outputStrength": "NONE"}
        ]
    },
    topicPolicyConfig={
        "topicsConfig": [
            {
                "name": "CompetitorDiscussion",
                "definition": "Discussions comparing our products to competitors",
                "type": "DENY"
            }
        ]
    },
    sensitiveInformationPolicyConfig={
        "piiEntitiesConfig": [
            {"type": "US_SOCIAL_SECURITY_NUMBER", "action": "BLOCK"},
            {"type": "EMAIL", "action": "ANONYMIZE"},
            {"type": "PHONE", "action": "ANONYMIZE"}
        ]
    }
)
