Building Bedrock Agents

What Are Bedrock Agents?

Bedrock Agents are autonomous AI workflows that can plan, execute multi-step tasks, and interact with external systems through action groups. They combine an LLM with tools and knowledge bases.

Agent Architecture

  • Foundation Model: The LLM brain that reasons and plans
  • Instructions: System prompt defining the agent's role
  • Action Groups: Lambda functions the agent can invoke
  • Knowledge Bases: RAG-enabled document stores for grounding
  • Guardrails: Safety filters and content policies

Creating an Action Group

Action groups connect your agent to external APIs via Lambda functions:

# Lambda function for a "lookup_order" action
def lambda_handler(event, context):
    action = event["actionGroup"]
    api_path = event["apiPath"]
    parameters = event["parameters"]

    order_id = next(p["value"] for p in parameters if p["name"] == "orderId")

    # Query your database
    order = db.get_order(order_id)

    return {
        "actionGroup": action,
        "apiPath": api_path,
        "response": {
            "body": json.dumps(order)
        }
    }

Agent Invocation Flow

When a user sends a query, the agent: (1) Analyzes the request, (2) Decides which actions to take, (3) Calls action groups or knowledge bases, (4) Synthesizes results into a response.
