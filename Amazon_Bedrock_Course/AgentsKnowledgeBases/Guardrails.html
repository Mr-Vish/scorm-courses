Guardrails for Responsible AI

What Are Bedrock Guardrails?

Guardrails are configurable safety filters that evaluate both user inputs and model outputs against your policies. They work with any Bedrock model.

Guardrail Components

  • Content Filters: Block harmful content by category (hate, violence, sexual, misconduct) with configurable thresholds (LOW/MEDIUM/HIGH)
  • Denied Topics: Define specific topics the model must refuse to discuss
  • Word Filters: Block specific words or phrases in input/output
  • Sensitive Information Filters: Detect and mask PII (names, emails, SSNs, credit cards)
  • Contextual Grounding: Detect hallucinations by checking if responses are grounded in provided context

Configuration Example

guardrail = bedrock.create_guardrail(
    name="CustomerServiceGuardrail",
    contentPolicyConfig={
        "filtersConfig": [
            {"type": "HATE", "inputStrength": "HIGH", "outputStrength": "HIGH"},
            {"type": "VIOLENCE", "inputStrength": "HIGH", "outputStrength": "HIGH"}
        ]
    },
    topicPolicyConfig={
        "topicsConfig": [
            {
                "name": "Investment Advice",
                "definition": "Providing specific investment recommendations",
                "type": "DENY"
            }
        ]
    }
)
