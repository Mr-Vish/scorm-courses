API Style Comparison

Spring AI Approach

@RestController
public class SpringAIController {
    
    private final ChatClient chatClient;
    
    public SpringAIController(ChatClient.Builder builder) {
        this.chatClient = builder
            .defaultSystem("You are a helpful assistant")
            .build();
    }
    
    @PostMapping("/chat")
    public String chat(@RequestBody String message) {
        return chatClient.prompt()
            .user(message)
            .call()
            .content();
    }
}

LangChain4j Approach

public interface Assistant {
    String chat(String message);
}

@RestController
public class LangChain4jController {
    
    private final Assistant assistant;
    
    public LangChain4jController() {
        ChatLanguageModel model = OpenAiChatModel.builder()
            .apiKey(System.getenv("OPENAI_API_KEY"))
            .build();
        
        this.assistant = AiServices.create(Assistant.class, model);
    }
    
    @PostMapping("/chat")
    public String chat(@RequestBody String message) {
        return assistant.chat(message);
    }
}

Function Calling Comparison

Spring AI

@Component
public class WeatherService {
    
    @Tool(description = "Get weather for a city")
    public String getWeather(String city) {
        return "Weather in " + city;
    }
}

// Auto-registered via Spring
ChatClient chatClient = builder
    .defaultFunctions("getWeather")
    .build();

LangChain4j

public class WeatherTools {
    
    @Tool("Get weather for a city")
    public String getWeather(String city) {
        return "Weather in " + city;
    }
}

// Explicit registration
Assistant assistant = AiServices.builder(Assistant.class)
    .chatLanguageModel(model)
    .tools(new WeatherTools())
    .build();

When to Choose Spring AI

  • Existing Spring Boot Applications: Seamless integration with Spring ecosystem
  • Convention Over Configuration: Prefer auto-configuration and minimal setup
  • Spring Expertise: Team is already familiar with Spring patterns
  • Enterprise Features: Need Spring Security, Actuator, Cloud integration
  • Rapid Development: Want to get started quickly with minimal boilerplate

When to Choose LangChain4j

  • Framework Flexibility: Not tied to Spring or want to use other frameworks
  • LangChain Familiarity: Team knows LangChain patterns from Python
  • Advanced Patterns: Need complex chains, agents, or custom workflows
  • Model Variety: Require support for more niche model providers
  • Explicit Control: Prefer explicit configuration over auto-configuration

Migration Considerations

Both frameworks can coexist in the same application:

@Configuration
public class HybridAIConfig {
    
    // Spring AI for main application
    @Bean
    @Primary
    public ChatClient springAIChatClient(ChatClient.Builder builder) {
        return builder.build();
    }
    
    // LangChain4j for specific use cases
    @Bean
    public Assistant langChain4jAssistant() {
        ChatLanguageModel model = OpenAiChatModel.builder()
            .apiKey(System.getenv("OPENAI_API_KEY"))
            .build();
        return AiServices.create(Assistant.class, model);
    }
}
