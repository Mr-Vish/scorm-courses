Getting Started with Amazon Q in AWS

What is Amazon Q in the AWS Console?

Amazon Q is an AI assistant built into the AWS Management Console that helps you understand AWS services, troubleshoot issues, and navigate resources. It is available directly in the console toolbar.

Key Capabilities

  • Service explanation: Ask about any AWS service and get instant explanations
  • Error diagnosis: Paste error messages and get troubleshooting guidance
  • Resource queries: Ask about your deployed resources and their configuration
  • Console navigation: Find the right console page for any task
  • CLI generation: Generate AWS CLI commands from natural language

Example Conversations

# Error diagnosis
User: "My Lambda function is returning AccessDeniedException when calling DynamoDB"
Q: "This error indicates your Lambda execution role doesn't have the required
    DynamoDB permissions. Here's how to fix it:
    1. Go to IAM > Roles > your-lambda-role
    2. Attach AmazonDynamoDBReadOnlyAccess (or a custom policy)
    3. Your Lambda will pick up the new permissions automatically"

# Resource queries
User: "How many EC2 instances are running in us-east-1?"
Q: "You have 12 EC2 instances running in us-east-1:
    - 8 t3.medium (production)
    - 3 t3.micro (development)
    - 1 m5.xlarge (database)"

Permissions and Access

Amazon Q in the Console respects your IAM permissions - it can only see and describe resources you have access to. No additional configuration is required beyond standard IAM roles.
