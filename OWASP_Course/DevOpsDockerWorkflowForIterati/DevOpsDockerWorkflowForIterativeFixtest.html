DEV OPS / Docker workflow for iterative fix/test

  1. When developing in VS Code, mount source directories into containers (-v) so edits are immediate.
  2. When you finish a change, build an image (for Java: mvn -DskipTests package then docker build .), run container and test.
  3. Keep DB persistence for testing across runs by mapping volumes for SQLite/H2 files or use in-memory but seed data on startup.
