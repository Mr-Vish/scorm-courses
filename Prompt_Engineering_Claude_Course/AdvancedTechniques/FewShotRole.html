Few-Shot and Role Prompting

Few-Shot Prompting

Provide Claude with examples of desired input-output pairs before your actual request. This teaches the pattern without fine-tuning.

<examples>
Input: "The app crashes on login"
Classification: Bug Report | Priority: High | Component: Auth

Input: "Add dark mode support"
Classification: Feature Request | Priority: Medium | Component: UI

Input: "Typo in README line 42"
Classification: Documentation | Priority: Low | Component: Docs
</examples>

Now classify this:
Input: "API returns 500 when uploading files over 10MB"

Role Prompting

Assigning Claude a specific role or persona focuses its knowledge and adjusts its communication style:

  • Expert roles: "You are a database performance engineer" focuses on optimization
  • Audience-aware roles: "Explain to a junior developer" adjusts complexity
  • Process roles: "You are a code reviewer at Google" applies specific standards

Combining Techniques

The most effective prompts combine multiple techniques:
