XML Tags and Structured Prompts

Why XML Tags?

Claude is specifically trained to understand XML tags as structural delimiters. They provide clear boundaries between different parts of your prompt, reducing ambiguity.

Common XML Tag Patterns

<instructions>
Analyze the following code for security vulnerabilities.
Focus on OWASP Top 10 categories.
</instructions>

<code>
public String getUser(String id) {
    return db.query("SELECT * FROM users WHERE id = " + id);
}
</code>

<output_format>
For each vulnerability found:
1. Category (e.g., SQL Injection)
2. Severity (Critical/High/Medium/Low)
3. Line number
4. Recommended fix with code example
</output_format>

Useful Tag Types

  • <context> - Background information
  • <instructions> - What Claude should do
  • <examples> - Few-shot examples
  • <constraints> - Rules and limitations
  • <document> - Source material to analyze
  • <output_format> - Desired response structure

Nesting Tags

Tags can be nested for complex prompts with multiple data sources or multi-step instructions.
