Transport Layers

stdio Transport

The default transport for local MCP servers. Communication happens over standard input/output (stdin/stdout):

  • Host launches the server as a subprocess
  • Messages are JSON-RPC 2.0 over stdin/stdout
  • Best for local tools, file system access, CLI integrations
  • No network configuration needed

SSE (Server-Sent Events) Transport

For remote servers accessible over HTTP:

  • Server exposes an HTTP endpoint
  • Client connects via SSE for server-to-client messages
  • Client sends messages via HTTP POST
  • Best for shared servers, cloud-hosted tools, team resources

Message Format

// Request (client to server)
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/call",
  "params": {
    "name": "query_database",
    "arguments": {
      "sql": "SELECT * FROM users LIMIT 10"
    }
  }
}

// Response (server to client)
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "content": [
      {"type": "text", "text": "Found 10 users..."}
    ]
  }
}
